# GLM 5.2 CPU Dense Model — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring up GLM 5.2 as a native-JAX, config-driven, dense-MLA model (`Glm4MoeLiteForCausalLM`, no DSA indexer) in `tpu-inference`, validated for mathematical correctness on CPU against inline torch / pure-`jnp` references.

**Architecture:** A new file `tpu_inference/models/jax/glm4_moe_lite.py` reuses DeepSeek-V3's MLA attention stack, MoE/router/dense-MLP modules, scaling-RoPE, `MLAEinsum` weight transform, decoder layer, and weight loader **by import**, but feeds them hyperparameters read from `hf_config` (à la `qwen3_moe.py`) instead of DeepSeek's hardcoded module globals. GLM deltas (sigmoid router with `e_score_correction_bias`, grouping off `n_group=topk_group=1`, `routed_scaling_factor=1.8` applied to the expert output, `norm_topk_prob=True`, silu only, `first_k_dense_replace` dense-then-MoE schedule, no per-head QK-norm) are encoded config-driven. MLA + the full forward use the production Pallas kernel on TPU, which has no CPU mode, so on CPU they are validated against a pure-`jnp` non-absorbed MLA reference.

**Tech Stack:** Python, JAX (CPU), Flax NNX, transformers (eager torch reference), pytest, pytest-mock. Test invocation: `JAX_PLATFORMS=cpu USE_DENSE_MOE=1 python -m pytest -s -v tests/models/jax/test_glm_v52.py`.

def _np_glm_router_ref(x_TD, gate_w_DE, bias_E, top_k, norm_topk_prob):
    """Numpy reference for the GLM router: sigmoid scoring, bias added only
    for top-k selection, weights gathered without bias, optional renorm."""
    logits = x_TD.astype(np.float32) @ gate_w_DE.astype(np.float32)
    probs = 1.0 / (1.0 + np.exp(-logits))           # sigmoid
    scored = probs + bias_E.astype(np.float32)       # bias before top-k
    idx = np.argsort(-scored, axis=-1)[:, :top_k]    # top-k by biased score
    w = np.take_along_axis(probs, idx, axis=-1)      # gather WITHOUT bias
    if norm_topk_prob:
        w = w / (w.sum(axis=-1, keepdims=True) + 1e-20)
    return w, idx

        x_TD, w1_EDF, w2_EDF, w3_EFD, weights_TX, idx_TX,

    """Numpy ref mirroring dense_moe_fwd + routed_scaling_factor-on-output
    + shared expert add (silu activation)."""

        return z / (1.0 + np.exp(-z))

    Mirrors DeepSeek's MHA-style kv_b_proj expansion (deepseek_v3.py:359-400)
    plus full causal O(n^2) attention. RoPE is the interleaved Deepseek

    def forward_reference(self, input_ids, positions):
        """CPU oracle full forward: pure-jnp MLA + dense-MoE, no Pallas kernel,
        using this model's loaded weights. Single sequence, causal."""
        import math
        from tpu_inference.layers.jax.moe.dense_moe import dense_moe_fwd
        from tpu_inference.layers.common.process_weights.moe_weights import \
            UnfusedMoEWeights
        from tpu_inference.models.jax.glm4_moe_lite import \
            mla_reference_nonabsorbed
        hf = self.vllm_config.model_config.hf_config
        x = self.embed_tokens(input_ids)
        qk_h = hf.qk_nope_head_dim + hf.qk_rope_head_dim
        if self.rope_emb.scaling_factor <= 1.0:
            yarn_mscale = 1.0
        else:
            yarn_mscale = 0.1 * hf.rope_scaling["mscale_all_dim"] * math.log(
                self.rope_emb.scaling_factor) + 1.0
        sm_scale = qk_h ** -0.5 * yarn_mscale ** 2
        for layer in self.layers:
            attn = layer.self_attn
            residual = x
            h = layer.input_layernorm(x)
            attn_out = mla_reference_nonabsorbed(
                h, positions=positions, rope=self.rope_emb,
                num_heads=hf.num_attention_heads,
                qk_nope_head_dim=hf.qk_nope_head_dim,
                qk_rope_head_dim=hf.qk_rope_head_dim,
                v_head_dim=hf.v_head_dim, kv_lora_rank=hf.kv_lora_rank,
                q_lora_rank=hf.q_lora_rank, q_a_proj=attn.q_a_proj,
                q_a_layernorm=attn.q_a_layernorm, q_b_proj=attn.q_b_proj,
                kv_a_proj_with_mqa=attn.kv_a_proj_with_mqa,
                kv_a_layernorm=attn.kv_a_layernorm, kv_b_proj=attn.kv_b_proj,
                o_proj=attn.o_proj, sm_scale=sm_scale)
            x = residual + attn_out
            residual = x
            h = layer.post_attention_layernorm(x)
            if isinstance(layer.mlp, DeepseekV3MLP):
                mlp_out = layer.mlp(h)
            else:
                moe = layer.mlp
                weights_TX, idx_TX = moe.gate(h)
                E = hf.n_routed_experts
                one_hot = jax.nn.one_hot(idx_TX, E, dtype=jnp.float32)
                full_TE = jnp.sum(one_hot * weights_TX[..., None], axis=1)
                mw = UnfusedMoEWeights(
                    w1_weight=moe.experts.kernel_gating_EDF.value,
                    w1_weight_scale=None, w1_bias=None,
                    w2_weight=moe.experts.kernel_up_proj_EDF.value,
                    w2_weight_scale=None, w2_bias=None,
                    w3_weight=moe.experts.kernel_down_proj_EFD.value,
                    w3_weight_scale=None, w3_bias=None)
                routed = dense_moe_fwd(
                    mw, h, jnp.float32,
                    P(ShardingAxisName.MLP_DATA, None), hf.hidden_act,
                    full_TE, self.mesh)
                routed = routed * hf.routed_scaling_factor
                mlp_out = routed + moe.shared_experts(h)
            x = residual + mlp_out
        return self.norm(x)

## Phase boundary note

This plan implements **Phase 0** (CPU dev env + runnable harness) and **Phase 1** (config-driven dense-MLA `Glm4MoeLiteForCausalLM`, no DSA), with per-submodule CPU numeric parity and a pure-`jnp` MLA/full-forward oracle. **Out of scope and deferred to follow-on plans:** Phase 2 (DSA lightning indexer + top-k + sparse-attn reference, gated by `index_topk`), Phase 3 (TPU bring-up: Pallas/Mosaic kernels, paged-KV, sharding, `AttentionMetadata.topk_indices`), MTP (`Glm4MoeLiteMTP`), quantization (fp8/fp4), and multi-batch/concurrent decode. The pure-`jnp` references built here (`mla_reference_nonabsorbed`, `Glm4MoeLiteModel.forward_reference`) are intended to become the kernel oracles in Phase 3.
