# Part A — the concepts, from primitives

## 0. The mental model: one residual stream of width 6144

A decoder-only transformer is a **residual stream**: a tensor `[batch, seq, 6144]`
that flows from embedding to final norm. Each layer **reads** the stream, computes a
delta, and **adds** it back (`x = x + sublayer(norm(x))`). Nothing is ever replaced.

```
ids [B,S]
 └─ embed_tokens        → x [B,S,6144]
 └─ 78 × DecoderLayer:
      x = x + Attention(RMSNorm(x))      #  MLA + DSA
      x = x + MLP(RMSNorm(x))            #  dense FFN (layers 0-2) or MoE (3-77)
 └─ final RMSNorm        → x [B,S,6144]
 └─ lm_head              → logits [B,S,154880]
```

`d_model = hidden_size = 6144` is the stream width — nearly every projection starts
or ends at 6144. The two exotic pieces vs a vanilla Llama: **(a)** attention is
**MLA + DSA**; **(b)** the FFN is a **256-expert MoE**. Everything else is standard.

- `q_b_proj` out = `H * qk_head_dim` = 64 × 256 = **16384**
- `kv_b_proj` out = `H * (qk_nope_head_dim + v_head_dim)` = 64 × 448 = **28672**

## 2. Embedding

`x = embed_tokens(input_ids)` — `nn.Embedding(154880, 6144)`, `[B,S]→[B,S,6144]`.
**`lm_head` is a separate `nn.Linear(6144, 154880)`, NOT tied** to the embedding
(`tie_word_embeddings=False`). `logits = lm_head(x)` at the end.

## 3. Norms (and the eps trap)

Two norm types. `GlmMoeDsaRMSNorm` (`:48`) — RMS only, fp32 upcast, learned `weight`,
**no bias, no mean-subtraction**:
```python
h = h.float(); var = h.pow(2).mean(-1, keepdim=True)
return weight * (h * rsqrt(var + eps)).to(in_dtype)
```
`nn.LayerNorm` — true LayerNorm (mean-subtract, **weight + bias**), used in exactly
one place: the DSA indexer's `k_norm`.

| Where | Class | dim | **eps** | bias? |
|---|---|---|---|---|
| `self_attn.q_a_layernorm` | RMSNorm | 2048 | **1e-6** ← class default, **not** `rms_norm_eps` | no |
| `self_attn.kv_a_layernorm` | RMSNorm | 512 | **1e-6** ← class default | no |
| `self_attn.indexer.k_norm` | **LayerNorm** | 128 | **1e-6** | **yes** |
| `input_layernorm` | RMSNorm | 6144 | 1e-5 | no |
| `post_attention_layernorm` | RMSNorm | 6144 | 1e-5 | no |
| `model.norm` (final) | RMSNorm | 6144 | 1e-5 | no |

**Trap:** `GlmMoeDsaRMSNorm.__init__(eps=1e-6)`; `q_a_layernorm`/`kv_a_layernorm` are
built *without* passing `rms_norm_eps`, so they silently use **1e-6**. The three
6144-wide stream norms explicitly pass **1e-5**. (vLLM uses 1e-5 for the two latent
norms — a real discrepancy. Match HF's 1e-6.)

## 4. RoPE from first principles (TWO conventions)

RoPE injects position by **rotating** 2D pairs of head dims by `position × θ_p`, with a
geometric frequency series over `dim = head_dim = 64` (that's why `__post_init__` sets
`head_dim = qk_rope_head_dim`):
```python
inv_freq[p] = 1 / theta**(2p/64)        # p=0..31  → 32 frequencies
freqs = position * inv_freq             # [..,32]
emb = cat(freqs, freqs)                 # [..,64]
cos, sin = emb.cos(), emb.sin()         # [B,S,64]
```
The rotation is unambiguous; **which two coords form a pair is a convention**, and GLM
uses both:

- **`rotate_half`** (NeoX) — **the DSA indexer.** Pairs `(i, i+32)`:
  ```python
  rotate_half(x) = cat(-x[...,32:], x[...,:32]);  q' = q*cos + rotate_half(q)*sin   # full 64-dim cos/sin
  ```
- **`apply_rotary_pos_emb_interleave`** (GPT-J/DeepSeek) — **MLA.** Pairs `(2p, 2p+1)`:
  ```python
  cos, sin = cos[...,:32], sin[...,:32]
  q1, q2 = q[...,0::2], q[...,1::2]
  q' = cat([q1*cos - q2*sin,  q2*cos + q1*sin], -1)
  ```

Same rotation, **different pairing of the 64 dims.** A weight matrix is trained for one
pairing (MLA's `q_b_proj`/`kv_a_proj` lay rope dims out interleaved; the indexer's
`wq_b`/`wk` lay them out half-split). Apply the wrong convention → you rotate the wrong
pairs → coherent garbage. You need **two** RoPE functions, each pinned bit-for-bit.
(On the real checkpoint the *indexer* convention is the one open HF-vs-vLLM divergence:
HF=rotate_half, vLLM=interleave, flag `indexer_rope_interleave=true`. MLA: both interleave.)

## 5. MLA — Multi-head Latent Attention (non-absorbed / HF form)

**Why:** vanilla MHA caches `64 heads × 256 × 2 = 32768` numbers/token/layer. MLA caches
a single **latent** of `kv_lora_rank + qk_rope_head_dim = 512 + 64 = 576`/token and
reconstructs per-head K/V on the fly (~57× smaller). HF *eager* materializes K/V anyway
(no cache win — it's the correctness oracle); the win is realized only by **absorption** (§6).

The forward (`GlmMoeDsaAttention.forward`, `:409`):
1. **Query** (compressed through rank 2048):
   `q_resid = q_a_layernorm(q_a_proj(x))` (6144→2048, RMSNorm eps **1e-6**);
   `q = q_b_proj(q_resid)` (2048→16384) → `[B,64,S,256]`; split **nope-first** `[192,64]`.
   `q_resid` is **also consumed by the indexer**.
2. **KV** (compressed to the 512 latent):
   `compressed_kv = kv_a_proj_with_mqa(x)` (6144→576); split `[512, 64]` = latent | rope-key.
   `kv_b_proj(kv_a_layernorm(latent))` (512→28672, RMSNorm eps **1e-6**) → `[B,64,S,448]`;
   split `[192, 256]` = k_nope | value.
3. **RoPE (interleave)** on `q_rot`/`k_rot`; the single 64-dim rope-key is shared across
   all heads (MQA-style), then broadcast to 64 heads.
4. Assemble per head: `query = [192 nope | 64 rope] = 256`, `key = [192 nope | 64 rope] = 256`.
5. `attn = softmax(q @ kᵀ * 256**-0.5 + mask) @ value`; `o_proj` (16384→6144).
   Scale = `qk_head_dim**-0.5 = 256**-0.5 = 0.0625` (× mscale² only under YaRN — not this ckpt).

## 6. MLA absorption (the cheap path vLLM/we run)

`k_nope_h = W_kbⁿ_h · c` where `c` = the 512 latent. So
`q_nope_h · k_nope_h = (q_nope_h · W_kbⁿ_h) · c`. **Absorb** `W_kbⁿ_h` into the query →
a 512-dim `q'_h` dotted directly against the cached 512 latent; absorb the value
up-proj into `o_proj`. You cache only **512 + 64 = 576/token** and never materialize
per-head K/V. Score = `(q_nope W_kbⁿ)·c  +  q_rope·k_rope`. This is the production kernel;
the non-absorbed §5 is the oracle. They must match numerically (the spec's *kernel-algebra*
gate — an absorption bug is invisible to the non-absorbed ref, hence its own injected-error test).

## 7. DSA — the lightning indexer (`GlmMoeDsaIndexer`, `:166`)

**Why:** even with MLA's cheap cache, attention is O(S²). DSA scores past tokens with a
cheap auxiliary head and keeps the **top `index_topk=2048`**; MLA then attends to only those.

Its own projections (separate from MLA): `wq_b`(2048→32·128), `wk`(6144→128, **one head**),
`k_norm`=LayerNorm(128, eps 1e-6), `weights_proj`(6144→32), scale `128**-0.5`.

Forward (`:197`), all the surprising bits:
- **rope-FIRST** split `[64,64]` (opposite of MLA's nope-first); **rotate_half** RoPE.
- `scores = relu( (q·kᵀ) * scale )` — **ReLU, no softmax**, all **fp32**.
- per-head importance `weights = weights_proj(x) * 32**-0.5`; `index_scores = Σ_h weights·scores`.
- **causal mask on the indexer's OWN scores, before top-k** (strict `key_pos > query_pos`).
- `topk_indices = index_scores.topk(min(2048, T)).indices` → `[B,S,topk]` int32.
- HF pins `indexer.weights_proj` to fp32 permanently; the optimized bf16 path **must**
  still score/reduce in fp32 and downcast only after top-k.
- The reference indexer uses a **Hadamard transform + fp8 scoring**; HF skips both because
  Hadamard is orthogonal (`Hq·Hk = q·k`) and fp8 is precision-only.

Back in attention, top-k becomes a mask: eager/sdpa scatter the 2048 indices into a
`[B,S,T]` bool and `-inf` the non-selected (combined with causal); the flash kernel
consumes the indices directly.

**The safety property** (drives all validation): `topk = min(2048, T)`, so for **S ≤ 2048
the indexer selects every valid token and DSA is bit-identical to dense attention.**
Below 2048 the DSA mask is a no-op (causal dominates); only past 2048 does it sparsify.

**Layer schedule** (`config.indexer_types[i]`): `"full"` layers run their own indexer (and
have indexer weights on disk); `"shared"` layers have `indexer is None` and **reuse the
previous full layer's `topk_indices`** (threaded via `prev_topk_indices`). Real GLM-5.2 is
mostly shared (full at 0,1,2 then every 4th). The indexer also keeps its **own** 128-dim
single-head key cache, distinct from MLA's 576-latent cache.

## 8. MoE (layers 3–77; `GlmMoeDsaMoE`, `:570`)

Router (`route_tokens_to_experts`, `:590`):
```python
logits = (x_fp32 @ gate.weight.T).sigmoid()          # SIGMOID, not softmax; fp32; [N,256]
choice = logits + e_score_correction_bias            # bias ONLY for selection
idx    = choice.topk(8).indices                      # n_group=1 ⇒ grouping is a no-op
w      = logits.gather(idx)                           # weights from BIAS-FREE sigmoid scores
w      = w / (w.sum(-1,keepdim=True)+1e-20) * 2.5     # norm_topk_prob, then × routed_scaling_factor
```
Experts: 256 SwiGLU FFNs, hidden 2048, fused `gate_up_proj [256, 4096, 6144]` /
`down_proj [256, 6144, 2048]`; run only each token's 8 experts; `down(silu(gate)*up)`,
weighted, summed. **Shared expert** (one SwiGLU, hidden 2048) runs on every token and is
added **un-scaled** on top.

Layers 0–2 use a plain `GlmMoeDsaMLP` (dense SwiGLU, hidden 12288) instead.

## 9. Decoder layer & model loop

```python
def DecoderLayer(x, ..., prev_topk_indices):
    h = input_layernorm(x)                                  # RMSNorm 6144 eps 1e-5
    attn, topk = self_attn(h, ..., prev_topk_indices)       # MLA + DSA
    x = x + attn
    h = post_attention_layernorm(x)                         # RMSNorm 6144 eps 1e-5
    x = x + mlp(h)                                          # dense or MoE
    return x, topk

# model:
x = embed_tokens(ids); cos,sin = rotary_emb(x, position_ids); causal = create_causal_mask(...)
topk = None
for layer in layers[:78]:
    x, topk = layer(x, causal, (cos,sin), position_ids, prev_topk_indices=topk)
x = norm(x); logits = lm_head(x)
```
`(cos,sin)` is built once and shared by MLA (interleave) and the indexer (rotate_half) —
same table, different pairing. `topk` threads across layers (the cross-layer sharing).

# Part B — concrete trace: `[1, 4, 6144]` through one decoder layer

Setup: **batch B=1, seq S=4**, real config dims. We trace a **MoE layer with a "full"
indexer** (e.g. layer 3). `T` = total cached length = 4 (prefill, fresh cache). Comments
give the **literal shape after each line**. Weight shapes are listed in the table at the end.

> Because `S=4 ≤ index_topk=2048`, the indexer's `topk = min(2048,4) = 4` selects **all**
> (causally-valid) tokens, so DSA degenerates to dense attention here. With e.g. `S=5000`
> the indexer would keep only the top 2048 keys per query and `-inf` the other ~2952.

### B.1 Attention sub-block

```python
x        = ...                                   # [1, 4, 6144]   input to the layer
residual = x                                     # [1, 4, 6144]
h        = input_layernorm(x)                    # RMSNorm(6144, eps 1e-5)        [1, 4, 6144]

# ---- MLA query path -------------------------------------------------------------
a        = q_a_proj(h)                           # Linear 6144→2048               [1, 4, 2048]
q_resid  = q_a_layernorm(a)                      # RMSNorm(2048, eps 1e-6)        [1, 4, 2048]   (← reused by indexer)
qb       = q_b_proj(q_resid)                     # Linear 2048→16384              [1, 4, 16384]
q_states = qb.view(1,4,64,256).transpose(1,2)    #                               [1, 64, 4, 256]
q_pass, q_rot = split(q_states, [192, 64], -1)   # NOPE-first         [1,64,4,192] , [1,64,4,64]

# ---- MLA KV path ----------------------------------------------------------------
ckv      = kv_a_proj_with_mqa(h)                 # Linear 6144→576                [1, 4, 576]
latent, k_rot = split(ckv, [512, 64], -1)        #                    [1,4,512] , [1,4,64]
ln       = kv_a_layernorm(latent)                # RMSNorm(512, eps 1e-6)         [1, 4, 512]
kb       = kv_b_proj(ln)                          # Linear 512→28672               [1, 4, 28672]
kb       = kb.view(1,4,64,448).transpose(1,2)    #                               [1, 64, 4, 448]
k_nope, value_states = split(kb, [192, 256], -1) #                    [1,64,4,192] , [1,64,4,256]

# ---- RoPE (interleave) ----------------------------------------------------------
# position_embeddings cos,sin:                                                    [1, 4, 64]
k_rot    = k_rot.view(1, 1, 4, 64)               #                               [1, 1, 4, 64]
q_rot, k_rot = apply_rotary_pos_emb_interleave(q_rot, k_rot, cos, sin)
                                                 # q_rot [1,64,4,64] , k_rot [1,1,4,64]
k_rot    = k_rot.expand(1, 64, 4, 64)            # broadcast 1 rope-key to 64 heads  [1,64,4,64]

query_states = cat([q_pass, q_rot], -1)          # [192 nope | 64 rope]           [1, 64, 4, 256]
key_states   = cat([k_nope, k_rot], -1)          # [192 nope | 64 rope]           [1, 64, 4, 256]
# cache.update(key_states, value_states)  → K,V cached as [1,64,T,256] each (HF eager materializes)

# ---- DSA indexer ("full" layer) -------------------------------------------------
iq   = wq_b(q_resid).view(1,4,32,128)            # Linear 2048→4096 → 32 heads    [1, 4, 32, 128]
iq_rot, iq_pass = split(iq, [64, 64], -1)        # ROPE-first         [1,4,32,64] , [1,4,32,64]
ik   = k_norm(wk(h)).unsqueeze(2)                # wk 6144→128, LayerNorm(128,1e-6)  [1, 4, 1, 128]
ik_rot, ik_pass = split(ik, [64, 64], -1)        #                    [1,4,1,64] , [1,4,1,64]
iq_rot, ik_rot = apply_rotary_pos_emb(iq_rot, ik_rot, cos, sin, unsqueeze_dim=2)   # ROTATE_HALF
iq   = cat([iq_rot, iq_pass], -1)                #                               [1, 4, 32, 128]
ik   = cat([ik_rot, ik_pass], -1).squeeze(2)     #                               [1, 4, 128]
# indexer_cache.update(ik) → ik becomes [1, T, 128] = [1, 4, 128]

scores = relu( matmul(iq.float(), ik.transpose(-1,-2).float().unsqueeze(1)) * 128**-0.5 )
         # iq [1,4,32,128] @ [1,1,128,4]  →                                       [1, 4, 32, 4]   (= [B,S,H,T])
w      = weights_proj(h).float() * 32**-0.5      # Linear 6144→32                 [1, 4, 32]
index_scores = matmul(w.unsqueeze(-2), scores).squeeze(-2)   # [1,4,1,32]@[1,4,32,4] [1, 4, 4]   (= [B,S,T])
index_scores = index_scores + indexer_causal_mask            # -inf future          [1, 4, 4]
topk_indices = index_scores.topk(min(2048,4)).indices        # = topk(4)           [1, 4, 4]  int32
# S=4 ≤ 2048 ⇒ all causally-valid tokens selected.

# ---- build the sparse mask (eager) and run attention ----------------------------
index_mask = ones(1,4,4, bool).scatter(-1, topk_indices, False).unsqueeze(1)      # [1, 1, 4, 4] (all False here)
mask       = causal_mask.masked_fill(index_mask, -inf)        # causal ∧ selected   [1, 1, 4, 4]

scores2 = query_states @ key_states.transpose(2,3) * 256**-0.5 # [1,64,4,256]@[1,64,256,4]  [1, 64, 4, 4]
scores2 = scores2 + mask                                       # broadcast [1,1,4,4]  [1, 64, 4, 4]
probs   = softmax(scores2, -1, dtype=fp32)                     #                     [1, 64, 4, 4]
ctx     = probs @ value_states                                # [1,64,4,4]@[1,64,4,256]  [1, 64, 4, 256]
ctx     = ctx.transpose(1,2).reshape(1, 4, 16384)             #                     [1, 4, 16384]
attn    = o_proj(ctx)                                          # Linear 16384→6144   [1, 4, 6144]

x = residual + attn                                            #                     [1, 4, 6144]
```

### B.2 MoE sub-block

```python
residual = x                                     #                               [1, 4, 6144]
h        = post_attention_layernorm(x)           # RMSNorm(6144, eps 1e-5)        [1, 4, 6144]

# ---- router -------------------------------------------------------------------
hf       = h.view(-1, 6144)                      # N = B*S = 4 tokens             [4, 6144]
logits   = (hf.float() @ gate.weight.T.float())  # gate.weight [256,6144]         [4, 256]
probs    = logits.sigmoid()                      #                               [4, 256]
choice   = probs + e_score_correction_bias       # [256] broadcast                [4, 256]
idx      = choice.topk(8, -1).indices            # n_group=1 ⇒ no grouping         [4, 8]
w        = probs.gather(1, idx)                  # bias-free sigmoid scores        [4, 8]
w        = w / (w.sum(-1, keepdim=True) + 1e-20) # norm_topk_prob                  [4, 8]
w        = w * 2.5                               # routed_scaling_factor           [4, 8]

# ---- experts (per token, run its 8) -------------------------------------------
# for each selected expert e: gate_up = hf_tok @ gate_up_proj[e].T  → [n,4096]; split → gate,up [n,2048]
#                             y = silu(gate)*up @ down_proj[e].T     → [n,6144]; × w
routed   = experts(hf, idx, w).view(1, 4, 6144)  #                               [1, 4, 6144]

# ---- shared expert (always on, unscaled) --------------------------------------
shared   = shared_experts(residual)              # SwiGLU hidden 2048             [1, 4, 6144]
mlp_out  = routed + shared                       #                               [1, 4, 6144]

x = residual + mlp_out                           # layer output                  [1, 4, 6144]
return x, topk_indices                           # topk threaded to the next layer
```

### B.3 The weights this layer touches (shapes)

| Param | Shape | Param | Shape |
|---|---|---|---|
| `q_a_proj` | [2048, 6144] | `kv_a_proj_with_mqa` | [576, 6144] |
| `q_a_layernorm` | [2048] | `kv_a_layernorm` | [512] |
| `q_b_proj` | [16384, 2048] | `kv_b_proj` | [28672, 512] |
| `o_proj` | [6144, 16384] | `input/post_attention_layernorm` | [6144] each |
| `indexer.wq_b` | [4096, 2048] | `indexer.wk` | [128, 6144] |
| `indexer.k_norm` | weight [128] + bias [128] | `indexer.weights_proj` | [32, 6144] |
| `mlp.gate.weight` | [256, 6144] | `mlp.gate.e_score_correction_bias` | [256] |
| `mlp.experts.gate_up_proj` | [256, 4096, 6144] | `mlp.experts.down_proj` | [256, 6144, 2048] |
| `mlp.shared_experts.{gate,up}_proj` | [2048, 6144] each | `mlp.shared_experts.down_proj` | [6144, 2048] |

(`nn.Linear(in,out)` stores weight `[out, in]`; in JAX you transpose to `x @ W[in,out]`.
A `"shared"` layer drops the four `indexer.*` rows and instead receives `prev_topk_indices`.)

### B.4 What changes at decode (S=1) and at long context (S>2048)

- **Decode** (`S=1`): every tensor above with an `S` axis becomes length 1; `T` grows to the
  full cached length. `query_states [1,64,1,256]`, `key_states/value_states [1,64,T,256]`,
  `index_scores [1,1,T]`, `topk_indices [1,1,min(2048,T)]`. The new token attends to all `T`
  cached keys; this is memory-bandwidth-bound, which is exactly why MLA (small cache) and DSA
  (only 2048 keys) exist.
- **Long context** (`S=5000`): `topk = 2048`, so `topk_indices` is `[1,5000,2048]` and the
  `index_mask` actually masks ~2952 keys/query — sparsity kicks in and the trace stops being
  dense-equivalent.

# Part C — hand-coding checklist (what silently breaks)

1. **Two RoPE functions.** MLA = interleave (pairs `2p,2p+1`, first-half cos/sin);
   indexer = rotate_half (pairs `i,i+32`, full cos/sin). RoPE table built on `head_dim=64`.
   Pin each bit-for-bit.
2. **Opposite split orders.** MLA q/k = **nope-first** `[192|64]`; indexer = **rope-first** `[64|64]`.
3. **Norm eps:** `q_a`/`kv_a_layernorm` = **1e-6** (class default); the three stream norms = **1e-5**;
   indexer `k_norm` = **LayerNorm** (mean-subtract + bias) eps 1e-6.
4. **Indexer:** ReLU, **no softmax**, all fp32; per-head weighted sum `×32**-0.5`; causal mask on
   its *own* scores before top-k.
5. **Scales:** MLA `256**-0.5` (qk_head_dim, not the 64 rope dim); indexer `128**-0.5`. No mscale (not YaRN).
6. **Router:** sigmoid; `e_score_correction_bias` for **selection only**; weights gathered bias-free;
   renorm; ×2.5; shared expert added **unscaled**.
7. **`q_resid` is shared** between MLA's query path and the indexer.
8. **dense == sparse for S ≤ 2048** — your single best correctness test (no reference model needed).
9. **`full`/`shared` schedule** — gate indexer weights on `indexer_types[i]`; thread `topk_indices`
   across layers; set `index_topk_freq>1` in any test config or you never exercise `shared`.
10. **Absorption** is the production MLA; keep a non-absorbed version as the oracle and check they
    match (absorption bugs hide there).
11. **Load:** drop `model.layers.78.*` (MTP); `lm_head` is **untied**; `kv_a_proj_with_mqa` concatenates
    `[512 | 64]`; split fused `kv_b_proj`→`k_up`/`v_up` and `gate_up_proj`→`gate`/`up`.

## Optimization landscape (HF oracle → vLLM GPU → TPU Pallas/Mosaic)

"Pallas" = JAX's kernel-authoring API; **"Mosaic"** = the TPU compiler backend that lowers
Pallas kernels to TPU code (the "M" thing). Writing a Pallas/Mosaic kernel = hand-writing a TPU kernel.

| Concern | HF (oracle) | vLLM (GPU) | Us (TPU, the spec) |
|---|---|---|---|
| MLA | non-absorbed, materializes K/V | absorbed flash-mla, paged | absorbed Mosaic `kernels/mla/v2` (+ jnp non-absorbed oracle) |
| Indexer | bf16/fp32, plain matmul, ReLU | fp8 `fp8_mqa_logits` + Hadamard; interleave | bf16, rotate_half (matches HF); fp8 deferred |
| Indexer `k_norm` eps | 1e-6 ✓ | 1e-5 (discrepancy) | 1e-6 |
| Sparse gather | dense `-inf` mask | flash consumes indices | Mosaic `sparse_attn`: one-hot gather, `-1` sentinel |
| MoE | Python per-expert loop | fused grouped GEMM | grouped-matmul (`gmm`), expert-axis sharded |
| Precision | fp32 | fp8 (UE8M0) | bf16/fp32 v1; fp8 roadmap |
| Parallelism | single process | TP/EP | `shard_map` over a mesh; sharded loader (S1 reshard hazard) |

Two TPU perf gotchas (spec §13): the ported one-hot sparse gather is **O(N), not O(topk)**
(reads every resident KV row) — production needs the **DMA scalar-prefetch** rewrite so cost
scales with 2048, not context length; and the existing tuned MLA kernel params don't match
GLM's 64-head/bf16 shapes, so it runs un-tuned defaults until GLM-shaped keys are added.
