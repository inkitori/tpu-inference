## §1. Authoritative architecture fact sheet (verified — DO NOT re-investigate)
Verified empirically (instantiated `GlmMoeDsaConfig()` + ran a CPU forward of `GlmMoeDsaForCausalLM`) and by reading real source: transformers 5.9.0 `glm_moe_dsa`, vLLM `deepseek_v2.py`, the `claude-deepseek-v4` fork.

### Config `GlmMoeDsaConfig` (defaults; real checkpoint overrides — read from `hf_config`)
hidden 6144 · 78 layers · dense-FFN `intermediate_size` 12288 · `moe_intermediate_size` 2048 · `n_routed_experts` 256 · `n_shared_experts` 1 · `num_experts_per_tok` **8** · `n_group`/`topk_group` **1/1** · `num_attention_heads` 64 · `num_key_value_heads` 64 · `q_lora_rank` 2048 · `kv_lora_rank` 512 · `qk_nope_head_dim` 192 · `qk_rope_head_dim` 64 · `v_head_dim` 256 · `qk_head_dim`(derived) 256 · `vocab_size` 154880 · `routed_scaling_factor` **2.5** · `norm_topk_prob` True · `rms_norm_eps` 1e-5 · `attention_bias` False · `tie_word_embeddings` False · `hidden_act` "silu".
- **RoPE key is `rope_parameters`** (dict), NOT `rope_scaling`. Default `{'rope_theta':10000.0,'rope_type':'default'}` (plain RoPE, **no YaRN**).
- **No `scoring_func` / `first_k_dense_replace` fields.** Scoring hardcoded **sigmoid**. Dense-vs-MoE from `mlp_layer_types` (default `["dense"]*min(3,L)+["sparse"]*(L-3)` → first 3 dense); vLLM reads `first_k_dense_replace` from checkpoint. **Read whichever the checkpoint supplies.**
- Indexer recompute-vs-reuse from `indexer_types` (default all `"full"`; `"shared"` reuses prev layer's top-k).
- `index_topk` 2048 · `index_head_dim` 128 · `index_n_heads` 32.
- **Sentinels raising AttributeError:** `rope_interleave`, `pretraining_tp`. Never reference.
- Extra `model.layers.78.*` = MTP head, **dropped on load** (`_keys_to_ignore_on_load_unexpected=[r"model\.layers\.78.*"]`).

### MLA attention (`GlmMoeDsaAttention`)
- q path: `q_a_proj`(6144→2048) → `q_a_layernorm`(RMSNorm 2048) → `q_b_proj`(2048→64·256=16384). Split **nope-first** `[192, 64]`; RoPE (non-interleaved) on the 64 slice.
- kv path: `kv_a_proj_with_mqa`(6144→512+64=576) → split `[512 | 64]`; `k_rope`=rope(last 64); `kv_lat`=`kv_a_layernorm`(RMSNorm 512); `kv_b_proj`(512→64·(192+256)=28672) → split per head `[k_nope 192 | v 256]`; `k`=cat(k_nope, broadcast k_rope).
- `o_proj`(16384→6144). **Softmax scale = `qk_head_dim**-0.5` = `256**-0.5`, NO mscale** under default rope.
- **No per-head q/k norm.**

### DSA indexer (`GlmMoeDsaIndexer`) — modeling lines 105-229
- `wq_b`(q_lora 2048 → n_heads·head_dim = 32·128 = 4096), **input is the shared q-LoRA residual** `q_a_layernorm(q_a_proj(x))` (NOT recomputed). reshape `[T,32,128]`, split **rope-first** `[64,64]`, rope the pe, recombine.
- `wk`(6144→128) → `k = k_norm(wk(hidden))`; **`k_norm`=LayerNorm(128, eps 1e-6, weight+bias)** applied to full 128 before split; split rope-first `[64,64]`, rope, recombine.
- `weights = weights_proj(hidden).float() * (n_heads**-0.5)` → `[T,32]`. `softmax_scale = head_dim**-0.5 = 128**-0.5`.
- `scores = einsum("shd,td->sht", q.f32, k.f32) * softmax_scale` → **`scores = F.relu(scores)`** (line **219**) → `index_scores = einsum("sht,sh->st", scores, weights)` → + causal mask → **NO softmax** → `topk = min(index_topk, T)`; `topk_indices = index_scores.topk(topk).indices`.
- Selection feeds attention as an **additive `-inf` mask** (scatter 0 at selected, -inf else) **combined with the causal mask by addition**; then standard MLA softmax. **No attn_sink** (that was the V4 fork).
- `"shared"` indexer layers reuse the previous layer's `topk_indices`.

### MoE / router (`GlmMoeDsaTopkRouter`, `GlmMoeDsaMoE`)
- **sigmoid** scoring (fp32 logits). `e_score_correction_bias` added to sigmoid scores **for selection only**; gathered `topk_weights` are from the **bias-free** sigmoid scores.
- group routing no-op when n_group=1 (all 256 eligible, pick top-8). `norm_topk_prob` → `/ (sum + 1e-20)`. Then `topk_weights *= routed_scaling_factor` (2.5).
- shared expert (`GlmMoeDsaMLP`, intermediate = `moe_intermediate_size · n_shared_experts`) **added unscaled**. silu gated everywhere.
- **`GlmMoeDsaNaiveMoe`** (modeling 499-536) = independent per-expert-loop MoE reference (use it as the **different-decomposition** oracle, not the production einsum).

### RoPE — definitive
**Non-interleaved (rotate-half/NeoX) for BOTH main attention and indexer.** Single function `apply_rotary_pos_emb` (modeling 67-102): `(x*cos)+(rotate_half(x)*sin)`, `rotate_half`=split-half. cos/sin built as `cat((freqs,freqs))`. ⚠️ **DSA docstrings claim "interleaved" — they are WRONG; trust the code.** The fork uses **complex-multiply** RoPE — does NOT transfer; build/force a rotate-half JAX rope and **pin it bit-for-bit vs HF** (highest porting risk).

### Weight names (HF checkpoint layout — map the UNFUSED names)
- Attn: `self_attn.{q_a_proj,q_a_layernorm,q_b_proj,kv_a_proj_with_mqa,kv_a_layernorm,kv_b_proj,o_proj}.weight` (`kv_a_proj_with_mqa` = `[512|64]` concat).
- Indexer: `self_attn.indexer.{wq_b,wk,weights_proj}.weight` + `self_attn.indexer.k_norm.{weight,bias}`.
- MoE: `mlp.gate.weight` + `mlp.gate.e_score_correction_bias`; **fused** `mlp.experts.gate_up_proj` (3-D `[E,2·moe_inter,hidden]` → split gate/up) + `mlp.experts.down_proj` (3-D `[E,hidden,moe_inter]`); `mlp.shared_experts.{gate,up,down}_proj.weight` (un-fused).
- Dense layers: `mlp.{gate,up,down}_proj.weight`.
- Top: `model.embed_tokens.weight`, `model.norm.weight`, `lm_head.weight` (**untied**). Drop `model.layers.78.*`.
- vLLM's `fused_qkv_a_proj`/`wk_weights_proj` are **internal post-load fusions** (`packed_modules_mapping`), NOT on disk — ignore for our loader.

## §3. Parity-harness methodology (the validation blueprint — adapt the fork's pattern)
Fork files: `claude-deepseek-v4/work/tpu-inference/tests/models/jax/test_deepseek_v4.py` + `_deepseek_v4_reference/`.
- **Identical weights both sides:** deterministically init the HF model (`normal(0,0.02)`, per-param dtype), copy its `state_dict` into the JAX model via **per-field converters** (JAX math is `x @ Wᵀ` → **no transposes** in the test path). Production loader uses a **regex name-map** (`_HF_TO_JAX_RULES` list of `(compiled_regex, path_template)`).
- **Tiered comparison** via `maxabs(a,b)=max|a_f32-b_f32|`, each submodule at its layer + full forward:
  | Check | Tol |
  |---|---|
  | RoPE (bit-for-bit) | exact/1e-6 |
  | Router | indices **exact** + weights 1e-2 |
  | RMSNorm/LayerNorm, dense FFN, embed/lm_head | 1e-3 |
  | MLA attention (absorbed & non-absorbed) | 1e-3 |
  | MoE experts (vs `GlmMoeDsaNaiveMoe`) | 1e-2 |
  | Indexer | top-k **set equality (exact)** + scores 1e-3 |
  | Full forward logits | 1e-3 (≈2e-3 long-ctx) + argmax agreement ≥0.95 |
- **CPU execution:** `JAX_PLATFORMS=cpu`; HF torch ref single-process (`WORLD_SIZE=1`, no `torch.distributed`); **mixed bf16/fp32 matching the real model** (NOT global fp32), upcast only inside `maxabs`; no `jax_enable_x64`.

## §5. HF-vs-vLLM discrepancies to keep as explicit Phase-3 risks (NOT v1 blockers)
1. **YaRN `mscale²`** — under `rope_type="deepseek_yarn"`, vLLM multiplies the attention scale by `mscale²` (`deepseek_v2.py:515`), HF DSA has **no mscale branch**. With **default rope both agree** (`yarn_get_mscale`→1.0). **v1 CPU tests use default rope.** Resolving the real-checkpoint case needs the real `config.json` (not on this machine) — top open question.

3. **GPU-only (skip for bf16/fp32 single-seq):** fp8 UE8M0 quant, DeepGEMM `fp8_mqa_logits`, `top_k_per_row` CUDA kernels, K-cache quant → replace with `jnp.top_k` + gather, functional RoPE.
