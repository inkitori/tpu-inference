## 0. Strategy (TPU-first) — read this

Develop and validate **on the real v6e from the start**, running the **real Pallas/Mosaic kernels** — not CPU stand-ins. It is driven by the prior **DeepSeek-V4** bring-up (`/home/enyouki/claude-deepseek-v4`), which was dominated by **multi-device, real-reshard-path bugs CPU structurally cannot reproduce** — chiefly **uninitialized-HBM-on-reshard ("S1")**: a device-side weight reshard read uninitialized HBM and baked per-process garbage into the weights, producing **coherent-looking-but-wrong** output at temp=0 (~30 sessions, root-cause hypothesis overturned repeatedly). This class does **not** appear single-device, and much of it doesn't appear under the CPU `interpret=True` path. So "develop on TPU" means **exercise the real kernels AND the real multi-device sharded weight-loader early** — not "run tiny tests on one chip."

**Three load-bearing guardrails the plan must keep:**
1. A pure-`jnp` fp32 reference **runnable on TPU** — the irreplaceable middle oracle. It runs on TPU not to improve math validation (math is device-independent) but because it is the **kernel's answer key**: the kernel only runs on TPU, so the reference must co-locate with it, else a CPU-vs-TPU fp32 reduction-order delta pollutes the one comparison meant to be clean.
2. **Staged triangulation** (§7) with a tight fp32 *math* gate distinct from a loose bf16 *kernel-noise* gate. Never collapse "kernel-on-TPU vs HF-eager" into one number.
3. **Multi-device** on the real v6e is a **required** gate (§7), not optional — and it must be **armed on a sharding geometry that actually triggers the S1 class** (§6 Phase 1b), not just any 4-chip run.

## 1. Goal & non-goals

### Goal
Bring up **GLM 5.2** — `GlmMoeDsaForCausalLM` (`model_type="glm_moe_dsa"`), i.e. **DeepSeek-V3.2-style MLA + DSA at GLM-5 scale** — as a **native-JAX** model in `tpu-inference`, **developed and validated on TPU with the real kernels from the start. DSA is implemented and validated, not deferred.**

The v1 finish line is **single-sequence correctness of the components on a scaled-down config, on TPU, single- and multi-device** — the dense MLA backbone, the MoE/router stack, and the DSA sparse path — validated against the in-tree `transformers GlmMoeDsaForCausalLM` eager forward via §7. **We do not run the full 78-layer model end-to-end in v1** (that and real-checkpoint generation are Phase 3+).

**Severable v1 fallback (schedule insurance):** if the Mosaic sparse kernel slips, v1 may ship as the **dense backbone (Phase 1) green single+multi-device, with DSA validated in jnp-ref form only** — indexer math + the §2 dense≡sparse equivalence + index-set parity — and the sparse *kernel* dropped to v1.1. The §2 equivalence is exactly what makes a kernel-free DSA milestone meaningful.

### Hardware & environment reality (prerequisite — see Phase 0)
- Host is **worker 0 of a 4-host v6e-16 slice** (`--deepsea_host_bounds=2,2,1`; 4 Trillium chips/host via `/dev/vfio/{0,1,2,3}`). ⚠️ **A single-process `jax.devices()` HANGS forever** — the SliceBuilder blocks waiting for the other 3 hosts (not contention; chips can be free; restarting `tpu-runtime` does not help). The **only** ways to get a multi-device surface: (a) run on **all 4 hosts together** (`gcloud compute tpus tpu-vm ssh --worker=all`, or Ray/Pathways), yielding all **16** chips; or (b) **reprovision a single-host `v6e-4`** (`host_bounds=1,1,1`) for a true 4-chip mesh. **There is no single-host 4-chip surface on this slice today** — resolve the test-surface choice in Phase 0 before Phase 1 (see CLAUDE.md).
- **Env:** system Python is 3.10 and **cannot** install the pinned `jax==0.10.1` (needs ≥3.11; repo Docker uses 3.12). No ready venv — Phase 0 stands one up. The persistent XLA **compilation cache is on by default** (`compilation_manager.py`), so edit→run is near-instant after the first cold compile.

### Non-goals (deferred past v1)
- **Full end-to-end model** (all 78 layers, real-checkpoint generation). Phase 3.
- **MTP** — the checkpoint's `model.layers.78` MTP head is **dropped on load** (`_keys_to_ignore_on_load_unexpected = [r"model\.layers\.78.*"]`).
- **Quantization** (fp8 UE8M0 / fp4). v1 runs bf16/fp32; the GPU DSA path is fp8 (DeepGEMM `fp8_mqa_logits`), v1 uses a bf16/fp32 indexer.
- **Aggressive kernel autotuning** — kernels are idiomatic-from-the-start; autotuning is bounded and post-correctness (§10).
- **Multi-batch / concurrent decode.** v1 is single-sequence (the fork has a **confirmed** `max_num_seqs>1` MoE-corruption bug — same uninit-HBM mechanism).

## 2. Architecture summary

GLM 5.2 is a decoder-only transformer where every layer is **MLA + DSA**, the FFN is **MoE** (after a few leading dense layers):

1. **MLA (Multi-head Latent Attention)** — cache one small **latent** (`kv_lora_rank=512` + shared `qk_rope_head_dim=64` rope key = 576 numbers) and **up-project** (`kv_b_proj`) to per-head K/V at attention time. Query path is compressed (`q_a_proj → q_lora_rank → q_a_layernorm → q_b_proj`). RoPE (**interleaved** — pairs adjacent dims, `apply_rotary_pos_emb_interleave`) applies only to the 64-dim rope slice for MLA; the indexer uses **rotate-half** (opposite convention, §2.2/§4.5). The attention splits **nope-first** `[qk_nope=192, qk_rope=64]`. Softmax scale = `qk_head_dim**-0.5 = 256**-0.5` (**no mscale** under default rope; §4.9). **No per-head QK-norm** — only `q_a_layernorm` and `kv_a_layernorm`.
2. **DSA (lightning indexer)** — a cheap auxiliary scorer ranks past tokens and selects the **top-`index_topk`=2048**; MLA then runs only over the selected set (additive `-inf` mask combined with the causal mask). V3.2/GLM form: per-head dot-product, **`relu`** on the scores, weighted head-sum, **no softmax**; `k_norm`=**LayerNorm** (the one LayerNorm in an otherwise RMSNorm model). The indexer splits **rope-first** `[64,64]` (opposite order to main attention) and **reuses the MLA query LoRA residual** `q_a_layernorm(q_a_proj(x))`. Some layers (`indexer_types[i]=="shared"`) **reuse the previous layer's top-k**.
3. **MoE** — 256 expert FFNs (`moe_intermediate_size=2048`) + 1 always-on shared expert; a router picks **top-8-of-256**. **sigmoid** scoring; `e_score_correction_bias` added **only for selection** (gathered weights use bias-free sigmoid scores); `norm_topk_prob` renormalizes; routed output scaled by `routed_scaling_factor` (default 2.5); the shared expert added **unscaled**. The first `first_k_dense_replace` (default 3) layers are a plain dense SwiGLU FFN.

**Key safety property (drives the validation ladder):** for any sequence of length ≤ `index_topk` (2048), `topk = min(index_topk, total_len)` selects **every** valid token, so DSA and dense attention produce the **identical** result. Sparsity only changes behavior past 2048 tokens. This lets us validate a correct **dense backbone** first (short seqs), then the **sparse path** (long seqs), with a clean equivalence check at the boundary — checkable **on TPU at fp32** (≈1e-6), isolating the entire sparse-plumbing bug class with no HF dependency.

## 3. Exact GLM 5.2 config (`GlmMoeDsaConfig`)

`GlmMoeDsaConfig` subclasses `Glm4MoeLiteConfig`; the generated config flattens inherited fields. **Defaults below verified live** via `GlmMoeDsaConfig()`. The real checkpoint's `config.json` is authoritative and overrides these; our config path reads everything from `hf_config`. DSA is gated by **`is_v32 = hasattr(config,"index_topk")`** — the `Glm4MoeLiteConfig` base has **no indexer**.

| Field | Default | Field | Default |
|---|---|---|---|
| `hidden_size` | 6144 | `n_routed_experts` | 256 |
| `num_hidden_layers` | 78 | `n_shared_experts` | 1 |
| `intermediate_size` (dense FFN) | 12288 | `num_experts_per_tok` | **8** |
| `moe_intermediate_size` | 2048 | `n_group` / `topk_group` | **1 / 1 (grouping OFF)** |
| `vocab_size` | 154880 | `routed_scaling_factor` | **2.5** |
| `max_position_embeddings` | 202752 | `norm_topk_prob` | True |
| `num_attention_heads` | 64 | scoring | **sigmoid (hardcoded; no `scoring_func` field)** |
| `num_key_value_heads` | 64 | `rms_norm_eps` | 1e-5 |
| `q_lora_rank` | 2048 | `attention_bias` | False |
| `kv_lora_rank` | 512 | `tie_word_embeddings` | False |
| `qk_nope_head_dim` | 192 | `qk_head_dim` (derived) | 256 |
| `qk_rope_head_dim` | 64 (`head_dim` attr → this) | `initializer_range` | 0.02 |
| `v_head_dim` | 256 | `hidden_act` | "silu" |

**RoPE config key is `rope_parameters`** (a dict), **not `rope_scaling`**. Default `{'rope_theta': 10000.0, 'rope_type': 'default'}` — **plain RoPE, no YaRN** by default. A real GLM-5 checkpoint may set `rope_type="deepseek_yarn"` with `factor`/`mscale_all_dim` (§4.9).

**Layer schedules** (two independent per-layer patterns, computed in `__post_init__`):
- **Dense-FFN vs MoE → `first_k_dense_replace` (default 3) → `mlp_layer_types`**: `first_k_dense_replace` **IS** a first-class HF config field (`configuration_glm_moe_dsa.py:128`) and the **primary driver** — `__post_init__` derives `mlp_layer_types = ["dense"]*min(first_k_dense_replace,L) + ["sparse"]*rest` (`:151-154`) ⇒ **first 3 dense**, rest MoE. Both keys always coexist; **read `first_k_dense_replace` directly** (or accept an explicit `mlp_layer_types` override if the checkpoint supplies one). vLLM keys on the same `first_k_dense_replace`.
- **Indexer recompute vs reuse → `indexer_types`**: computed in `__post_init__` (`configuration_glm_moe_dsa.py:135-146`) from `index_topk_pattern` (explicit `'F'`/`'S'` string) if present, else from `index_topk_freq` (default 1) **and** `index_skip_topk_offset` (default 2) via `'full' if (max(i-offset+1,0) % freq)==0 else 'shared'`. ⚠️ **With the default `index_topk_freq=1`, every layer is `"full"`** (no `"shared"`) — so the scaled-down test config **must** set `index_topk_freq>1` (or an explicit `index_topk_pattern`) to produce ≥1 `"shared"` layer and exercise the reuse path. A `"shared"` layer reuses the previous layer's `topk_indices` (`skip_topk`). **Every attention layer is DSA-sparse**; "full"/"shared" only controls whether the *indexer scores* are recomputed. **Read from `hf_config`**.

**DSA indexer fields:** `index_topk=2048`, `index_head_dim=128`, `index_n_heads=32`, `indexer_types` (computed). All read from `hf_config`; never hardcoded.

**Sentinels (DSA removes these → `AttributeError`):** `rope_interleave`, `pretraining_tp`. Never reference them.

**Norms — definitive:**
- `q_a_layernorm` = RMSNorm(`q_lora_rank=2048`, **eps 1e-6** — `GlmMoeDsaRMSNorm` class default, **NOT** `rms_norm_eps`; `modeling_glm_moe_dsa.py:377,:49`); `kv_a_layernorm` = RMSNorm(`kv_lora_rank=512`, **eps 1e-6**; `:385`). ⚠️ **HF-vs-vLLM disagree here** — vLLM uses `rms_norm_eps` (1e-5) for these two; **match HF's 1e-6**.
- `input_layernorm`, `post_attention_layernorm`, final `model.norm` = RMSNorm(`hidden_size`, eps `rms_norm_eps`=1e-5) — the **only** norms that use `rms_norm_eps`.
- (indexer) `k_norm` = **LayerNorm**(`index_head_dim=128`, eps **1e-6**) — has **weight AND bias**. The only LayerNorm in the model.
- **No per-head q/k norm.**

## 4. GLM/DSA deltas vs the existing DeepSeek-V3 JAX path (the things to get exactly right)

The repo's `models/jax/deepseek_v3.py` is **plain DeepSeek-V3 (no indexer)**; its module globals are already close to GLM (`routed_scaling_factor=2.5`, `num_experts_per_token=8`, `first_k_dense_replace=3`, 256 experts). Each delta below is a known foot-gun.

1. **Config-driven hparams** — replace DeepSeek's hardcoded module globals (`deepseek_v3.py:76-108`) with values from `hf_config` (à la `qwen3_moe.py`). GLM differs in `n_group` (1 vs 8), `hidden_size` (6144 vs 7168), `q_lora_rank` (2048 vs 1536), `qk_nope_head_dim` (192 vs 128), `v_head_dim` (256 vs 128), `num_attention_heads` (64 vs 128), and adds the `index_*` fields.
2. **`n_group = topk_group = 1`** — grouping **off**; take top-8 from all 256. `DeepSeekV3Router.get_topk_indices` already skips the group branch when `n_groups==1`.
3. **Router gate in fp32**, **sigmoid** scoring, `e_score_correction_bias` added **before top-k for selection only**; **gathered weights use bias-free sigmoid scores**; `norm_topk_prob` renorm (`/ (sum + 1e-20)`).
4. **`routed_scaling_factor` (default 2.5) applied to the routed-expert output**, **not** to router logits; shared expert added **unscaled**. Site already correct in `SharedFusedMoe.__call__`; pass the config value through `DeepseekV2Moe`.
5. **RoPE — MLA is INTERLEAVED, the indexer is rotate-half.** **The single highest porting risk.** MLA uses `apply_rotary_pos_emb_interleave` (pairs adjacent even/odd dims; `modeling_glm_moe_dsa.py:434`, fn `:302-338`); the indexer uses rotate-half `apply_rotary_pos_emb` (`:239`, HF's own comment at `:238` confirms it). ⚠️ **HF and vLLM AGREE** (vLLM MLA `is_neox_style=False` = interleaved; vLLM indexer `is_neox_style=True` = rotate-half) — there is **no** HF-vs-vLLM RoPE divergence. For MLA, port the **interleaved** layout (the repo's `DeepseekScalingRotaryEmbedding` is interleaved and now matches MLA, modulo YaRN scaling); use the base rotate-half `RotaryEmbedding` only for the **indexer**. Pin **two** bit-for-bit tests: MLA vs `apply_rotary_pos_emb_interleave`, indexer vs `apply_rotary_pos_emb` (§7). Main attention splits **nope-first** `[192,64]`; indexer splits **rope-first** `[64,64]`.
6. **silu only** for FFN/expert activation (gated SwiGLU).
7. **Dense-then-MoE schedule** — `first_k_dense_replace` **IS** an HF config field (default 3; `configuration_glm_moe_dsa.py:128`) and the **primary driver**: HF derives `mlp_layer_types` from it in `__post_init__` (`:151-154`). Read `first_k_dense_replace` directly; accept an explicit `mlp_layer_types` override if the checkpoint supplies one (§3). vLLM keys on the same field.
8. **No per-head QK-norm** (§3).
9. **Two distinct softmax scales — don't conflate them.** The **MLA attention** scale = `qk_head_dim**-0.5 = 256**-0.5` (`modeling_glm_moe_dsa.py:398`); the **DSA indexer** scale is separate = `index_head_dim**-0.5 = 128**-0.5 ≈ 0.0884` (`:195`), applied fp32 at `:246`. Under **YaRN**, HF multiplies the **MLA** scale by `mscale²` (`:398-404`) — and **vLLM does the identical thing** (`deepseek_v2.py`), so HF and vLLM **AGREE** (no longer an open divergence). The mscale branch touches **only** the MLA scale; the **indexer scale is a fixed `128**-0.5`, no mscale ever**. Under default rope both leave the bare MLA scale. v1 tests use default rope; whether the real `config.json` enables YaRN is the only remaining open item (§9).
10. **DSA indexer = V3.2/GLM form, NOT the fork's V4 form.** Keep: q-up-proj from the shared q-LoRA residual, **`relu` on scores**, per-head `weights_proj × n_heads**-0.5`, head-sum, `top_k`. **Drop** (V4-only): the **compressor/CSA** (GLM's K is direct `k_norm(wk(hidden_states))` per-token — the single biggest math change), SWA flavors, mHC sinkhorn, APE. `k_norm` is **LayerNorm**, not RMSNorm.
11. **PyTorch quirks — nothing to port:** do **not** carry over DeepSeek-V3's layer-0 residual `.clone()` or fp16-overflow rescale; the HF GLM-DSA port already dropped them (verified — `DecoderLayer.forward` is a plain residual add).

## 5. Code structure & reuse map (TPU-forward)

**New file `tpu_inference/models/jax/glm_moe_dsa.py`, reusing DeepSeek modules by import.**

> **TPU-forward notes:** (a) Implement **two** MLA paths and run **both on TPU**: a **pure-`jnp` non-absorbed `GlmMoeDsaAttention`** (the fp32 *oracle*/answer key, §7) and the **production absorbed MLA Pallas kernel** (`kernels/mla/v2`). Do **not** reuse the base rotate-half class for MLA. v1 uses default rope (no YaRN). (c) The production **`JaxAutoWeightsLoader` regex loader** is brought up **early** (not deferred), because the most expensive prior bug (S1, uninit-HBM-on-reshard) lives in the **multi-device sharded weight-load path** — the multi-device gate (§6, §7) needs it.

**Reuse as-is (import) from `deepseek_v3.py`** (re-locate by symbol; line hints are 2026-06-19): `DeepseekV3MLA` (587), `DeepseekV2Moe` (846), `DeepSeekV3Router` (989), `DeepseekV3MLP` (738), `SharedFusedMoe` (815)/`JaxMoE` (`moe/moe.py:129`), `DeepseekV3DecoderLayer` (942), `MLAEinsum` (488). 3. **RoPE wiring** — interleaved for MLA, rotate-half for the indexer — + GLM router constants.
4. **DSA sparse-attention kernel (Phase 2)** — **port the fork's already-wired Mosaic `sparse_attn` kernel** and adapt: drop `attn_sink`, drop the compressor (direct `k_norm(wk(x))`), keep the one-hot-matmul gather (the fork hit `E2003 UnprovenMemoryAccessAlignment` on v6e and fixed it this way, commit `ace9d576`), `-1` sentinel = ignore. ⚠️ The kernel's in-file docstring says "not yet wired" but that is **stale** — the fork wired it into both prefill/decode call sites via `shard_map` (commit `c2557e26`); **port that wiring, don't reinvent it.** The kernel is still **non-paged / reads all N**, so give it the paged/flattened KV surface.

**Surgical, behavior-preserving param-izing of DeepSeek modules:** `DeepseekV2Moe.__init__` (and the `SharedFusedMoe(...)` block, incl. the `expert_axis_name` global — `deepseek_v3.py:908`) read ~10 module globals. ## 7. Testing & validation strategy

**Oracle = the in-tree `transformers GlmMoeDsaForCausalLM` eager forward** (torch, CPU, random weights, no download) — an **independent** implementation of every §4 delta. It runs on the host; the model-under-test runs on TPU; comparison is a **host-side `maxabs`** (JAX arrays → host numpy). Upcast both sides to fp32 **inside** `maxabs` (a bf16 TPU output returns as numpy bf16 — load-bearing). No `jax_enable_x64`.

### The triangulation (the core of the strategy)
Three implementations, three single-delta comparisons — never a fused HF-vs-bf16-kernel verdict:

```
HF-eager (CPU, fp32)  ──[MATH gate, 1e-3]──►  jnp-ref (TPU, fp32)  ──[KERNEL gate]──►  Pallas kernel (TPU)
        independent oracle                      our answer key                          shipped path
```

- **MATH gate** (HF-fp32 vs jnp-ref-fp32-on-TPU, rtol≈1e-3): catches every §4 math/spec bug (RoPE convention, head splits, `256**-0.5` scale, kv_b split, router bias-for-selection, routed_scaling 2.5, relu placement). This is the gate with *teeth* — prove it by an **injected-1%-error test** (perturb a scale, assert the fp32 gate catches it). Without an fp32 reference on TPU there is **no** math gate: the bf16 kernel noise floor (~4e-3/element, compounding over depth) is larger than a real ~1% bug.
- **KERNEL gate** (jnp-ref vs kernel, on TPU): run the kernel **twice** — (a) fp32 vs jnp-ref-fp32 at tight tolerance to check kernel *algebra* — **cast the `q`/`ql_nope` inputs to fp32** (the kernel's fp32 output path is gated on `q_dtype`, `kernel.py:1938`; `:370` asserts output dtype == q_dtype), in addition to `s_dtype=fp32, p_same_dtype_as_v=False`; (b) bf16 shipped defaults vs jnp-ref-bf16 at the **empirically-measured bf16 floor** (run the jnp-ref in bf16 to derive the floor — don't guess). Repo precedent: bf16 attention ~0.1–0.2 (`ragged_paged_attention_kernel_v2_test.py:129-134`, `mla_v2_test.py:405`); MoE/elementwise ~5e-2.
  - ⚠️ The kernel gate carries **double duty**: the non-absorbed jnp-ref deliberately does NOT absorb, so a bug in the **kv_b absorption** algebra shows up only here and the jnp-ref can't catch it. Give the **fp32 kernel-algebra check the same teeth** as the math gate: a **second injected-error test that perturbs the kv_b absorption split** and asserts the fp32 kernel gate catches it.

### Tiered comparison (each submodule at the layer where it lives, plus full forward)
| Check | Tolerance / method |
|---|---|
| RoPE — MLA interleaved, indexer rotate-half (two tests) | exact / 1e-6 |
| Router | top-k indices **exact** (fp32 gate, well-separated scores) + weights 1e-2 |
| RMSNorm / LayerNorm, dense FFN, embed/lm_head | 1e-3 (fp32 math gate) |
| MLA attention | math gate 1e-3 (jnp-ref vs HF); kernel-algebra 1e-3 fp32 (+ absorption injected-error test); shipped bf16 at measured floor |
| MoE experts (vs **eager** `GlmMoeDsaNaiveMoe` — assert `config._experts_implementation == "eager"`) | 1e-2 |
| Indexer (`"full"` layers; absent on `"shared"`) | **boundary-aware tie-tolerant** top-k set equality + index_scores 1e-3 fp32 |
| Dense==sparse (§2, seq ≤ topk) | **fp32 ≈1e-6 on TPU** — isolates sparse plumbing, no HF dependency |
| Full forward | per-layer index-**set** equality (tie-tolerant) + fp32 math gate 1e-3 / bf16 shipped at floor + argmax ≥0.95 |

### Multi-device (required gate, not optional)
The bug class that motivated TPU-first lives here. On the **4-chip mesh**, via the **real sharded weight-loader**:
- **N-device fp32 result == 1-device fp32 result** (sharding must be value-invariant). Compare 1/2/4-device.
- **NaN-poison HBM** before weight load; assert no NaN/garbage leaks into outputs (catches the S1 uninit-HBM-on-reshard read; the fix is host-stacking into the sharded layout, not device reshard — §5, where `gmm_v2(zero_initialize=True)` is noted as disproven).
- Run the gate against the **sharding-geometry stress fixture** (§6 Phase 1b), not just the tiny config, or it may pass vacuously. Exercise `expert_axis_name` partitioning. **A single-device v6e pass is NOT TPU validation.**

### Determinism & ties
TPU is seeded-reproducible but **not bit-equal** to the torch ref; reductions are non-associative and order differs (more so multi-device). Top-k ties are a documented hazard (`fused_moe_v1_test.py:299-300`, jax #34620). Hence: the fp32 math gate uses well-separated scores where exact equality is asserted; the indexer set compare is **boundary-aware tie-tolerant** (adjudicate boundary swaps by recomputed fp32 scores); `index_scores` gated separately at 1e-3 so a relu/scale bug shows up even when the set survives. Pin the torch ref: fixed seeds, single-process (`WORLD_SIZE=1`), fp32, eager.

## 10. Kernel optimization — idiomatic now, bounded tuning later

Performance is pursued in two modes; neither is a continuous autonomous loop during correctness bring-up.

**Now (free, during bring-up):**
- Write kernels in the **performant TPU idiom from day 1** (sensible block/tile structure, avoid pathological layouts, `shard_map`-wrapped Mosaic, host-side RoPE-freq precompute).
- **Reuse existing tuned MLA params** via `get_tuned_params` (`tuned_params.py`) where shapes match.
- **One-time structural perf smoke for the DSA sparse kernel** at **real `index_topk=2048`** before its kernel spec is frozen — a single "is the one-hot-gather structure within an order of magnitude of viable?" measurement (not a sweep), so a structural cliff is caught while a rewrite is still cheap. (MLA needs no such smoke — its structure is proven.)

**Later (bounded, post-correctness):** *autotuning* starts only **after single-sequence correctness is green and the kernel spec is frozen** (notably the YaRN `mscale²` question, §9). Then:
- Run **bounded, human-triggered sweeps** at **real GLM-5.2 dims** (tiny-config perf is meaningless — the serving lookup is exact-shape-keyed) using the existing tuner (`tools/kernel/tuner/v1`, `--run_locally=True`).
- For DSA, **write a dedicated `sparse_attn` tuner** (subclass `KernelTunerBase`) once the kernel is correct — the existing tuner is MLA-decode-only with no feedback loop.
- **Guardrails (mandatory):** a tuned config is accepted only if the **full correctness suite still passes**; a **tuned-vs-baseline regression test** (≤5% slower, modeled on `mla_tuned_vs_baseline_test.py:131-135`, extended to GLM/DSA shapes); tuning runs against a **frozen kernel commit**; apply-back is **human-reviewed**, never silently mutated.
- **Stop criterion:** discrete bounded sweeps — stop when the best config is within timing noise (<3–5%) of the previous best, or at a fixed TPU-hour budget.
