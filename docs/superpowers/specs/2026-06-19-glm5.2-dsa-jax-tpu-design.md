# GLM 5.2 (DSA) on JAX/TPU — Design Spec

**Date:** 2026-06-19
**Status:** Approved design (reworked) — ready for implementation planning (`writing-plans`).
**Supersedes:** an earlier spec that targeted the WRONG class — `Glm4MoeLiteForCausalLM` = GLM-4.7-Flash (MLA-only, no DSA, interleaved RoPE, 1.8 scale). That correction is captured in §1.1; the old spec file has been removed (recoverable from git history).
**Companion docs:** `../research/2026-06-19-glm5.2-jax-tpu-research-findings.md` (file-reference grounding) and `../research/2026-06-19-glm5.2-explainer.md` (plain-language background). The architecture-identity rework that produced this spec is summarized in §1.1.

All architecture facts below were re-derived from the **real source** in this rework (transformers 5.9.0 `glm_moe_dsa`, vLLM `deepseek_v2.py`, the `claude-deepseek-v4` fork), and the key ones were verified **empirically** by instantiating `GlmMoeDsaConfig()` and running a CPU forward of `GlmMoeDsaForCausalLM`.

## 1. Goal & non-goals

### Goal
Bring up **GLM 5.2** — architecture `GlmMoeDsaForCausalLM` (`model_type="glm_moe_dsa"`), i.e. **DeepSeek-V3.2-style MLA + DeepSeek Sparse Attention (DSA) at GLM-5 scale** — as a **native-JAX** model in `tpu-inference`. Correctness is established **on CPU first** against an authoritative in-tree reference, then on TPU. **DSA is implemented and validated, not deferred.** The v1 finish line is **single-sequence correctness**: CPU parity (dense MLA backbone *and* the DSA sparse path) vs the in-tree `transformers GlmMoeDsaForCausalLM` eager forward, then single-sequence correctness on TPU.

### 1.1 What changed vs the superseded spec (read this first)
The prior spec targeted `Glm4MoeLiteForCausalLM`. That class is **GLM-4.7-Flash**, not GLM 5.2. Concretely corrected here:
- **Class/identity:** `GlmMoeDsaForCausalLM` / `glm_moe_dsa`, config `GlmMoeDsaConfig(Glm4MoeLiteConfig)`. The Lite class has **no indexer at all**.
- **Scale:** GLM-5 (hidden **6144**, **78** layers, **256** experts, q_lora **2048**, **64** heads), not Lite ~30B (2048/47/64).
- **RoPE:** **non-interleaved (NeoX / rotate-half)** for *both* main MLA and the indexer. DSA removes Lite's `rope_interleave` (it raises `AttributeError`). The prior `rope_interleave=True` was Lite behavior — **wrong for 5.2**. ⚠️ The DSA module docstrings *claim* "interleaved"; the **code is non-interleaved** — trust the code.
- **Indexer ReLU:** the indexer **applies `F.relu`** to the scores (`modeling_glm_moe_dsa.py:219`). The prior "no ReLU" claim was wrong.
- **`routed_scaling_factor`:** default **2.5** (not 1.8 — 1.8 was a Lite value). Config-driven; the real checkpoint is authoritative.
- **`num_experts_per_tok`:** **8** (not 4).
- **Layer schedule fields:** there is **no `scoring_func` / `first_k_dense_replace` field** on `GlmMoeDsaConfig`. Dense-vs-MoE comes from `mlp_layer_types` (HF) / `first_k_dense_replace` (vLLM/checkpoint); indexer recompute-vs-reuse from `indexer_types`. Scoring is hardcoded **sigmoid**.
- **Oracle:** the in-tree `transformers GlmMoeDsaForCausalLM` eager class (runs on CPU, random weights, no download), not hand-transcribed numpy refs. The prior plan's numpy "references" were **tautological** (re-derived the same math) — replaced.

### Non-goals (explicitly deferred past v1)
- **MTP** (multi-token prediction). The checkpoint carries an extra `model.layers.78` MTP head that is **dropped on load** (`_keys_to_ignore_on_load_unexpected = [r"model\.layers\.78.*"]`). Not built in v1.
- **Quantization** (fp8 UE8M0 / fp4). v1 runs bf16/fp32. The GPU DSA path is fp8 (DeepGEMM `fp8_mqa_logits`); v1 uses a bf16/fp32 indexer.
- **Performance / kernel tuning.** v1 needs *correct*, not fast.
- **Multi-batch / concurrent decode.** v1 is single-sequence. (The fork has a **confirmed** `max_num_seqs>1` MoE-corruption bug — out of v1 scope.)

These are listed so the implementation plan does not silently pull them in.

## 2. Architecture summary

GLM 5.2 is a decoder-only transformer where every layer is **MLA + DSA**, the FFN is **MoE** (after a few leading dense layers):

1. **MLA (Multi-head Latent Attention)** — cache one small **latent** (`kv_lora_rank=512` + `qk_rope_head_dim=64` shared rope key = 576 numbers) and **up-project** (`kv_b_proj`) to per-head K/V at attention time. Query path is compressed (`q_a_proj → q_lora_rank=2048 → q_a_layernorm → q_b_proj`). RoPE (non-interleaved) applies only to the 64-dim rope slice; the attention splits **nope-first** `[qk_nope=192, qk_rope=64]`. Softmax scale = `qk_head_dim**-0.5 = 256**-0.5` (**no mscale** under default rope; see §4.9). **No per-head QK-norm** — only `q_a_layernorm`(RMSNorm 2048) and `kv_a_layernorm`(RMSNorm 512).
2. **DSA (lightning indexer)** — a cheap auxiliary scorer ranks all past tokens and selects the **top-`index_topk`=2048**; MLA attention then runs only over the selected set (implemented as an additive `-inf` mask combined with the causal mask). V3.2/GLM form: per-head dot-product, **`relu`** on the scores, weighted head-sum, **no softmax**; `k_norm` = **LayerNorm** (eps 1e-6, the one LayerNorm in an otherwise RMSNorm model). The indexer splits **rope-first** `[qk_rope=64, rest=64]` (opposite order to main attention) and **reuses the MLA query LoRA residual** `q_a_layernorm(q_a_proj(x))` (it does not recompute q-LoRA). Some layers (`indexer_types[i]=="shared"`) **reuse the previous layer's top-k** instead of recomputing.
3. **MoE** — 256 expert FFNs (`moe_intermediate_size=2048`) + 1 always-on shared expert; a router picks **top-8-of-256**. **sigmoid** scoring; `e_score_correction_bias` added **only for selection** (the gathered weights use the bias-free sigmoid scores); `norm_topk_prob` renormalizes; routed-expert weights are then scaled by `routed_scaling_factor` (default 2.5); the shared expert is added **unscaled**. The first `first_k_dense_replace` (default 3) layers are a plain dense SwiGLU FFN (`intermediate_size=12288`).

**Key safety property (drives the phase plan):** for any sequence of length ≤ `index_topk` (2048), `topk = min(index_topk, total_len)` selects **every** valid token, so DSA and dense attention produce the **identical** result. Sparsity only changes behavior past 2048 tokens. This lets us validate a correct **dense backbone** first (Phase 1, short sequences) and then the **sparse path** (Phase 2, long sequences) with a clean equivalence check at the boundary.

## 3. Exact GLM 5.2 config (`GlmMoeDsaConfig`)

`GlmMoeDsaConfig` subclasses `Glm4MoeLiteConfig` (`modular_glm_moe_dsa.py:81`); the generated config flattens inherited fields. **Defaults below were verified live** via `GlmMoeDsaConfig()`. The real checkpoint's `config.json` is authoritative and overrides these; our config path reads everything from `hf_config`.

| Field | Default | Field | Default |
|---|---|---|---|
| `hidden_size` | 6144 | `n_routed_experts` | 256 |
| `num_hidden_layers` | 78 | `n_shared_experts` | 1 |
| `intermediate_size` (dense FFN) | 12288 | `num_experts_per_tok` | **8** |
| `moe_intermediate_size` | 2048 | `n_group` / `topk_group` | **1 / 1 (grouping OFF)** |
| `vocab_size` | 154880 | `routed_scaling_factor` | **2.5** |
| `max_position_embeddings` | 202752 | `norm_topk_prob` | True |
| `num_attention_heads` | 64 | scoring | **sigmoid (hardcoded; no `scoring_func` field)** |
| `num_key_value_heads` | 64 | `rms_norm_eps` | 1e-5 |
| `q_lora_rank` | 2048 | `attention_bias` | False |
| `kv_lora_rank` | 512 | `tie_word_embeddings` | False |
| `qk_nope_head_dim` | 192 | `qk_head_dim` (derived) | 256 |
| `qk_rope_head_dim` | 64 (`head_dim` attr → this) | `initializer_range` | 0.02 |
| `v_head_dim` | 256 | `hidden_act` | "silu" |

**RoPE config key is `rope_parameters`** (a dict), **not `rope_scaling`**. Default `rope_parameters = {'rope_theta': 10000.0, 'rope_type': 'default'}` — **plain RoPE, no YaRN** by default. A real GLM-5 checkpoint may set `rope_type="deepseek_yarn"` with `factor`/`mscale_all_dim`; see §4.9 for the mscale consequence.

**Layer schedules** (two independent per-layer patterns, computed in `__post_init__`):
- **Dense-FFN vs MoE → `mlp_layer_types`**: if absent, `["dense"]*min(3,L) + ["sparse"]*(L-3)` ⇒ **first 3 layers dense**, rest MoE. (Lite was first-1-dense.) vLLM reads the equivalent `config.first_k_dense_replace` from the checkpoint. **Our config path must read whichever the checkpoint supplies** (prefer `first_k_dense_replace` if present, else derive from `mlp_layer_types`).
- **Indexer recompute vs reuse → `indexer_types`**: if absent, all layers `"full"` (default `index_topk_freq=1`). A `"shared"` layer reuses the previous layer's `topk_indices` (`skip_topk`). **Every attention layer is DSA-sparse**; "full"/"shared" only controls whether the *indexer scores* are recomputed.

**DSA indexer fields:** `index_topk=2048`, `index_head_dim=128`, `index_n_heads=32`, `indexer_types` (computed). All read from `hf_config`; never hardcoded.

**Sentinels (DSA disables these — they raise `AttributeError`):** `rope_interleave`, `pretraining_tp`. Never reference them.

**Norms — definitive:**
- `q_a_layernorm` = RMSNorm(`q_lora_rank=2048`, eps `rms_norm_eps`)
- `kv_a_layernorm` = RMSNorm(`kv_lora_rank=512`, eps `rms_norm_eps`)
- `input_layernorm`, `post_attention_layernorm`, final `model.norm` = RMSNorm(`hidden_size`)
- (indexer) `k_norm` = **LayerNorm**(`index_head_dim=128`, eps **1e-6**) — has **weight AND bias**. The only LayerNorm in the model.
- **No per-head q/k norm.**

## 4. GLM/DSA deltas vs the existing DeepSeek-V3 JAX path (the things to get exactly right)

The repo's `models/jax/deepseek_v3.py` is **plain DeepSeek-V3 (no indexer)**; its module globals are already close to GLM (`routed_scaling_factor=2.5`, `num_experts_per_token=8`, `first_k_dense_replace=3`, 256 experts). The deltas to implement are below; each is a known foot-gun.

1. **Config-driven hparams** — replace DeepSeek's hardcoded module globals (`deepseek_v3.py:76-108`) with values read from `hf_config` (à la `qwen3_moe.py`). GLM differs from the DeepSeek globals in: `n_group` (GLM **1** vs 8), `hidden_size` (6144 vs 7168), `q_lora_rank` (2048 vs 1536), `qk_nope_head_dim` (192 vs 128), `v_head_dim` (256 vs 128), `num_attention_heads` (64 vs 128), and adds the `index_*` fields.
2. **`n_group = topk_group = 1`** — grouping **off**; take top-8 from all 256. `DeepSeekV3Router.get_topk_indices` already skips the group branch when `n_groups==1` (`deepseek_v3.py:1057`).
3. **Router gate in fp32**, **sigmoid** scoring, `e_score_correction_bias` added **before top-k for selection only**; the **gathered weights use bias-free sigmoid scores**; `norm_topk_prob` renorm (`/ (sum + 1e-20)`).
4. **`routed_scaling_factor` (default 2.5) applied to the routed-expert output** (equivalently, to the normalized top-k weights), **not** to router logits; shared expert added **unscaled**. Site already correct in `SharedFusedMoe.__call__:830` (`final *= routed_scaling_factor`, then unscaled shared add) — pass the config value through `DeepseekV2Moe`.
5. **Non-interleaved (NeoX / rotate-half) RoPE for BOTH** main MLA and the indexer. This is the **single highest porting risk** (the fork uses complex-multiply RoPE; DeepSeek-V3's rope convention must be confirmed/forced to rotate-half for GLM). Pin it bit-for-bit vs HF `apply_rotary_pos_emb` (§7 rec C). Main attention splits **nope-first** `[192,64]`; indexer splits **rope-first** `[64,64]`.
6. **silu only** for FFN/expert activation (gated SwiGLU).
7. **Dense-then-MoE schedule** from `first_k_dense_replace`/`mlp_layer_types` (config-driven, default first-3-dense).
8. **No per-head QK-norm** (see §3).
9. **DSA softmax scale = `qk_head_dim**-0.5 = 256**-0.5` with NO mscale** under default rope. ⚠️ **HF-vs-vLLM discrepancy:** under a YaRN checkpoint, **vLLM** multiplies the scale by `mscale²` (`yarn_get_mscale(factor, mscale_all_dim)²`, `deepseek_v2.py:515`), but **HF DSA has no mscale branch at all**. With default rope both agree (`yarn_get_mscale`→1.0). v1 CPU tests use **default rope** (no ambiguity). The YaRN `mscale²` is an explicit **Phase-3 reconciliation risk** requiring the real `config.json` (§9, §10).
10. **DSA indexer = V3.2/GLM form, NOT the fork's V4 form.** Keep: q-up-proj from the shared q-LoRA residual, **`relu` on scores**, per-head `weights_proj × n_heads**-0.5`, head-sum, `top_k`. **Drop** (V4-only): the **compressor/CSA** (GLM's K is direct `k_norm(wk(hidden_states))` per-token — the single biggest math change), SWA flavors, mHC sinkhorn, APE. `k_norm` is **LayerNorm**, not RMSNorm.
11. **PyTorch quirks to skip in JAX:** layer-0 residual `.clone()`, fp16-overflow rescale (we run bf16/fp32; the rescale is `dtype==float16`-gated anyway).

## 5. Code structure (approved decision)

**New file `tpu_inference/models/jax/glm_moe_dsa.py`, reusing DeepSeek modules by import.**

> **Implementation-rework note (2026-06-19, from the plan's adversarial review — verified against real source):** two reuse assumptions below did **not** survive contact with the code and are corrected here. (a) **`mla_attention` is TPU-only** (Mosaic/Pallas, no XLA/CPU path), so `DeepseekV3MLA.compute_attention` cannot run on CPU; v1 CPU uses a **pure-`jnp` `GlmMoeDsaAttention`** (non-absorbed) written in the new file, and reuses `DeepseekV2Moe`/`DeepSeekV3Router`/`DeepseekV3MLP` (which run on CPU via the `DENSE_MAT` backend). The absorbed MLA + kernel is Phase 3. (b) **RoPE has no flag**: the repo's `DeepseekScalingRotaryEmbedding` is hard-coded *interleaved*; GLM is rotate-half. v1 uses **default rope (no YaRN)**, so write a self-contained rotate-half rope (matching the base `RotaryEmbedding` + HF). (c) The production **`JaxAutoWeightsLoader` regex loader** loads real checkpoints (which run on TPU) → Phase 3; v1 validates a per-field converter + a coverage golden test. `weight_utils.py` lives at `models/jax/utils/weight_utils.py`.

- **Import & reuse** from `deepseek_v3.py`: `DeepseekV2Moe`, `DeepSeekV3Router`, `DeepseekV3MLP`, `SharedFusedMoe`/`JaxMoE`, the `MLAEinsum` split math (algebra only — its `load_weights` asserts `quant_config is not None`, so write a standalone no-quant `kv_b` split for CPU); for v1 CPU, **do not** reuse `DeepseekV3MLA`/`DeepseekV3DecoderLayer` (TPU-kernel-bound) — write GLM-specific equivalents. Use `JaxRmsNorm`/`JaxLayerNorm` (`norm.py`), `JaxEinsum`/`JaxLmHead` (`linear.py`), `JaxEmbed` (`embed.py`), `UnquantizedConfig({})` (`quantization/unquantized.py`). RoPE: self-contained rotate-half (not the interleaved `DeepseekScalingRotaryEmbedding`). Production loader (`JaxAutoWeightsLoader` + `LoadableWithIterator`, `models/jax/utils/weight_utils.py`) = Phase 3.
- **Write GLM-specific deltas only:**
  1. **Config-driven hparams** read from `hf_config`, replacing DeepSeek's module globals.
  2. **DSA `GlmMoeDsaIndexer` module + top-k selection** (the genuinely new piece; V3.2/GLM form per §3/§4).
  3. **Non-interleaved RoPE** wiring + the GLM router constants.
- **Surgical, behavior-preserving param-izing of DeepSeek modules:** `DeepseekV2Moe.__init__` (and the `SharedFusedMoe(...)` block, incl. the `expert_axis_name` global — `deepseek_v3.py:908`) read ~10 module globals. Add optional kwargs defaulting to the current globals so existing DeepSeek behavior is **unchanged** (existing DeepSeek tests stay green as a gate). Prefer optional args over rewrites.
- **DSA hook point:** in v1 CPU, the indexer + additive `-inf` mask insert into the new pure-`jnp` `GlmMoeDsaAttention.__call__` (Phase 1 runs it dense via a `dsa=False` flag; Phase 2 turns on `dsa=True`). On TPU (Phase 3) the equivalent hook is `DeepseekV3MLA.compute_attention` (`deepseek_v3.py:663`) immediately before `mla_attention(...)` (`:702`), threading `topk_indices` into the kernel.
- **Registration:** `_MODEL_REGISTRY["GlmMoeDsaForCausalLM"] = GlmMoeDsaForCausalLM` in `models/common/model_loader.py`, with the inline-import pattern (`:73-100`); add the arch to `_PP_DISABLED_MODELS` (`:60-61`). If a checkpoint declares a custom `auto_map` arch string, map it to the same class.

**Weight names** (HF checkpoint layout, prefix `model.layers.{i}.`) — our loader maps the **unfused HF names** (vLLM's `fused_qkv_a_proj`/`wk_weights_proj` are *internal post-load* fusions, NOT on disk):
- **Attention:** `self_attn.{q_a_proj, q_a_layernorm, q_b_proj, kv_a_proj_with_mqa, kv_a_layernorm, kv_b_proj, o_proj}.weight` — `kv_a_proj_with_mqa` concatenates `[kv_lora_rank(512) | qk_rope_head_dim(64)]`; reuse DeepSeek's `MLAEinsum` split of `kv_b_proj`→`k_up`/`v_up`.
- **Indexer:** `self_attn.indexer.{wq_b, wk, weights_proj}.weight` + `self_attn.indexer.k_norm.{weight, bias}` (LayerNorm ⇒ has bias).
- **MoE (sparse layers):** `mlp.gate.weight` + `mlp.gate.e_score_correction_bias`; **fused** experts `mlp.experts.gate_up_proj` (3-D `[E, 2·moe_inter, hidden]`, **split into gate/up**) and `mlp.experts.down_proj` (3-D `[E, hidden, moe_inter]`); `mlp.shared_experts.{gate_proj, up_proj, down_proj}.weight` (**un**fused).
- **Dense layers (0..first_k_dense_replace-1):** `mlp.{gate_proj, up_proj, down_proj}.weight` (un-fused).
- **Top-level:** `model.embed_tokens.weight`, `model.norm.weight`, `lm_head.weight` (**untied**). Drop `model.layers.78.*` (MTP).

## 6. Phase plan

### Phase 0 — CPU dev environment + HF oracle
**Deliverable:** JAX-CPU setup + a runnable harness whose oracle is the in-tree `transformers GlmMoeDsaForCausalLM` eager forward.
- Isolated venv: `jax[cpu]`, `flax`, `transformers` (≥5.9.0, ships `glm_moe_dsa`), `torch`, `numpy`, `pytest`, `pytest-mock`. **Omit `libtpu`** (no macOS wheel; `requirements.txt` hard-pins it). **Do NOT `pip install -e .`** (pulls the pin + the AVX2 XLA flag). Run with `JAX_PLATFORMS=cpu` and scrub `--xla_cpu_max_isa=AVX2` from `XLA_FLAGS` (injected unconditionally by `env_override.py:17`; breaks XLA on arm64).
- Stand up `tests/models/jax/test_glm_moe_dsa.py` with a **tiny real `GlmMoeDsaConfig`** (scaled-down dims, ≥1 dense + ≥1 MoE layer, `index_topk` small enough to exercise both ≤topk and >topk regimes, **default rope**), the **HF-eager oracle fixture** (build `GlmMoeDsaForCausalLM.from_config(cfg, attn_implementation="eager")` + deterministic init), a `t2j` weight-copy helper, and a `maxabs` comparison helper (per the fork blueprint, §7).
- **Exit criterion:** `JAX_PLATFORMS=cpu pytest test_glm_moe_dsa.py` collects; the **HF oracle runs a finite forward** on the tiny config on CPU (proves the oracle exists and runs).

### Phase 1 — Dense-MLA GLM backbone (vs HF eager, ≤ index_topk)
**Deliverable:** a complete, registered, config-driven `glm_moe_dsa.py` whose **dense backbone** (MLA + MoE + router + RoPE, indexer present but trivially selecting all tokens on short sequences) matches HF eager. De-risks ~80% of the model.
- Param-ize `DeepseekV2Moe`/`SharedFusedMoe` (incl. `expert_axis_name`) behavior-preservingly (§5).
- **Pin RoPE bit-for-bit** (non-interleaved) vs HF `apply_rotary_pos_emb` (§4.5, rec C) — its own unit test.
- Per-submodule parity vs HF eager, **identical weights loaded into both sides** (rec A): router (top-k indices exact, weights 1e-2), RMSNorms + LayerNorm, dense FFN, MoE experts (vs HF `GlmMoeDsaNaiveMoe`, an **independent** per-expert decomposition — rec D), embedding/lm_head, MLA attention (1e-3).
- **Run the production absorbed MLA on CPU** via a no-quant CPU loader (rec B) so the *shipped* path is the *validated* path; assert absorbed ≡ non-absorbed and both ≡ HF.
- **Weight-mapping golden test:** load the HF `state_dict` into the JAX model via the converter; assert every param maps (no `KeyError`, no leftover random-init), fused `gate_up_proj` split correct, indexer names present, `layers.78` dropped.
- **Exit criterion:** full forward on the tiny config, seq < `index_topk`, **JAX logits match HF eager at rtol≈1e-3 fp32** (+ argmax agreement ≥0.95).

### Phase 2 — DSA (lightning indexer + sparse selection), vs HF eager
**Deliverable:** the DSA indexer + top-k selection (V3.2/GLM form) **built and validated on CPU**, plus the pure-`jnp` sparse reference that becomes the TPU kernel oracle.
- Implement `GlmMoeDsaIndexer` (§3/§4.10): `wq_b` from the shared q-LoRA residual, `wk`+`k_norm`(LayerNorm), `weights_proj`, rope-first split, non-interleaved RoPE, `relu` on scores, weighted head-sum, `top_k=min(index_topk, T)`. Support `indexer_types=="shared"` (reuse previous top-k).
- Wire the indexer into `compute_attention` as an **additive `-inf` mask** combined with the causal mask (GLM has **no attn_sink**; sparse attention is standard MLA softmax over the masked set).
- **Indexer parity vs HF `GlmMoeDsaIndexer`:** top-k **set equality (exact)** + `index_scores` at 1e-3, identical weights.
- **Validation ladder vs HF eager:** (1) seq ≤ `index_topk`: sparse output == dense output (§2 equivalence) **and** top-k contains all valid tokens; (2) seq > `index_topk`: full-forward JAX logits match HF eager at ≈1e-3 (+ argmax ≥0.95) and the selected index set matches HF.
- **Exit criterion:** both ladder checks pass vs HF eager on CPU.

### Phase 3 — TPU bring-up (separate follow-on plan; needs TPU hardware)
**Deliverable:** single-sequence GLM 5.2 correctness on TPU.
- Port/adapt the fork's Mosaic `sparse_attn` kernel (single shared KV head, in-kernel one-hot gather, `topk_idxs` `-1`=ignore, no causal-in-kernel); extend MLA paged attention to take gather indices; add an **indexer-key KV cache** (raw per-token keys, à la HF `_cached_keys`); add `topk_indices` to `AttentionMetadata`.
- **Reconcile the YaRN `mscale²` HF-vs-vLLM discrepancy** (§4.9) against the real `config.json` and vLLM-GPU; reconcile vLLM's fp8 indexer path (ReLU location, fp8 `mqa_logits`).
- The pure-`jnp` references from P1/P2 become the **kernel oracles**.
- **Exit criterion:** single-sequence generation on TPU matches the CPU reference within bf16 tolerance. (Perf, fp8, and the multi-batch MoE bug are post-v1.)

## 7. Testing & validation strategy

**Oracle = the in-tree `transformers GlmMoeDsaForCausalLM` eager forward** (verified CPU-runnable with random weights, no download). This is an **independent** implementation of every §4 delta — it kills the circular-self-oracle problem of the superseded plan (whose numpy refs re-derived the code's own math, proving only `jnp==numpy`). Methodology is the `claude-deepseek-v4` fork's parity harness, adapted:

- **Identical weights both sides:** deterministic random init of the HF model (e.g. `normal(0, 0.02)`, per-param dtype), then copy its `state_dict` into the JAX model via per-field converters. **Correction (verified):** the JAX math here is `x @ kernel` with `kernel` shaped `(in,out)`, while HF `nn.Linear` is `(out,in)` — so the converter **must transpose** every linear (`embed_tokens` is the exception; `lm_head` transposes). Also randomize **buffers** not just `named_parameters()` — `e_score_correction_bias` is a *buffer*, and if left at 0 the bias-for-selection router delta passes vacuously. The production **regex name-map / `JaxAutoWeightsLoader`** is Phase 3 (loads real TPU checkpoints).
- **Tiered comparison** (per the fork's calibrated tolerances), each submodule compared at the layer where it lives, plus the full forward:
  | Check | Tolerance |
  |---|---|
  | RoPE (rec C, bit-for-bit) | exact / 1e-6 |
  | Router | top-k indices **exact** + weights 1e-2 |
  | RMSNorm / LayerNorm, dense FFN, embed/lm_head | 1e-3 |
  | MLA attention (non-absorbed jnp ref vs HF; + absorption-algebra check) | 1e-3 |
  | MoE experts (vs **eager** `GlmMoeDsaNaiveMoe` — force `experts_implementation="eager"`) | 1e-2 |
  | Indexer | top-k **set equality** (tie-tolerant) + index_scores 1e-3 (recomputed from HF submodules — HF's indexer returns indices only) |
  | Full forward logits | per-layer index-**set** equality (primary, tie-tolerant) + logits 1e-3/~2e-3 gated on set-match + argmax ≥0.95 |
- **CPU execution:** `JAX_PLATFORMS=cpu`; HF torch ref single-process (`WORLD_SIZE=1`, no `torch.distributed`); **mixed bf16/fp32 matching the real model** (not global fp32) — upcast only inside `maxabs`. No `jax_enable_x64`.
- **Instantiation:** `MockVllmConfig` + tiny `GlmMoeDsaConfig`, single-device mesh from `conftest.py` (`assert num_devices==1`). The existing DeepSeek **attention** test hard-requires a TPU (`TestDeepseekV3Attention.setUp`) — our CPU tests must not inherit that.
- **Adversarial hardening folded in (from a 4-agent adversarial review):**
  - **A.** HF eager is the full+submodule oracle with identical weights (above).
  - **B.** *(Adapted — `mla_attention` is TPU-only, so the production paged kernel cannot run on CPU.)* On CPU validate (1) a pure-`jnp` non-absorbed MLA vs HF at 1e-3, (2) the **absorption algebra** — a no-quant `kv_b` split into `k_up`/`v_up` reconstructs the same per-head K/V as the direct projection (the split math `deepseek_v3.py:540-543` is trivial without dequant; `MLAEinsum.load_weights` itself stays asserting `quant_config is not None` and is untouched in v1). The shipped paged kernel is validated on TPU (Phase 3).
  - **C.** Pin JAX RoPE **bit-for-bit** vs HF (highest-risk shared component; GLM = rotate-half, the fork's complex-multiply does NOT transfer). Write a self-contained rotate-half rope — the repo's `DeepseekScalingRotaryEmbedding` is interleaved.
  - **D.** Use a **different decomposition** than the code under test — HF's per-expert `GlmMoeDsaNaiveMoe` loop. **Caveat (verified):** `@use_experts_implementation` defaults to a fused `grouped_mm` on CPU (close to the JAX einsum); force `experts_implementation="eager"` (assert `config._experts_implementation == "eager"`) to get the genuine independent loop.
  - **E.** Replace the broken DeepSeek regression gate (the stale `test_deepseek_v3.py:40` imports a **non-existent** `DeepSeekV3WeightLoader` → collection error) with **construction tests**: GLM override (256 experts / `n_group=1` / top-8) vs DeepSeek defaults (256 / 8-group / top-8), asserting the param-izing didn't regress DeepSeek and didn't silently build GLM against a DeepSeek global (e.g. `expert_axis_name`).

**What CPU correctness buys / doesn't:** matching HF eager on CPU kills the dominant new-model bug class (weight mapping/fusion, router logic, MLA/MoE math, RoPE/head-split/scale, indexer top-k) — ~70-80% of "does it work". It does **not** catch (Phase-3): Pallas/Mosaic kernel bugs, sharding/uninit-HBM reshard, paged-KV/runtime `start_pos`/metadata threading, bf16/fp8 drift, the YaRN `mscale²` HF-vs-vLLM question, or the multi-batch MoE bug. Those are chased on TPU — which is exactly why the pure-`jnp` references are written to be reused as kernel oracles.

## 9. Risks & mitigations

| Risk | Mitigation |
|---|---|
| **YaRN `mscale²` HF-vs-vLLM divergence** (§4.9): HF DSA omits it; vLLM applies it under `deepseek_yarn` | v1 CPU uses **default rope** (both agree). Phase-3 task: read the real `config.json`, replicate HF's exact scale on CPU, reconcile vs vLLM-GPU. Documented as the top open question (§10). |
| **RoPE convention** — GLM is rotate-half/NeoX; the fork is complex-multiply; DeepSeek-V3's rope may be interleaved | Build/force a **non-interleaved** JAX rope; **pin bit-for-bit vs HF** as its own test (rec C). Do not copy the fork's RoPE. |
| Indexer **ReLU** discrepancy — HF + fork apply `relu`; vLLM's fp8 kernel path "passes as-is" | **Apply relu** (match HF, the authoritative CPU oracle + the fork). Flag vLLM's opaque fp8 kernel as a Phase-3 check. |
| **Weight fusion mismatch** — real checkpoint has unfused q_a/kv_a + fused expert `gate_up_proj`; vLLM's `fused_qkv_a_proj` is internal | Loader maps **unfused HF names**, splits `gate_up_proj`; **weight-mapping golden test** asserts full coverage (no KeyError / leftover random init). |
| Param-izing DeepSeek globals regresses DeepSeek (esp. `expert_axis_name`) | Optional args defaulting to current globals; **construction regression tests** (rec E); keep DeepSeek tests green. |
| Production **absorbed MLA never runs on CPU** (`assert quant_config is not None`) | No-quant CPU loader (rec B); assert absorbed ≡ non-absorbed ≡ HF. |
| **CPU-invisible TPU bugs** (kernel, sharding/uninit-HBM reshard, runtime `start_pos`/metadata, fp4→fp8, multi-batch MoE) | Out of v1 CPU scope; enumerated for Phase 3; pure-`jnp` refs become kernel oracles. Single-seq only in v1. |
| Indexer `index_n_heads`/`index_head_dim` differ in a real checkpoint | All indexer dims read from `hf_config`; never hardcode. |
| Stale `test_deepseek_v3.py` imports non-existent `DeepSeekV3WeightLoader` (collection error) | Do **not** depend on that file for the regression gate; use construction tests (rec E). |
