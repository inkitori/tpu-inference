**Status:** Approved design — TPU-forward rework. Ready for implementation planning (`writing-plans`).

## 0. Why TPU-first (the strategy change, read this)

We develop and validate **on the TPU (v6e-16) from the start**, running the **real Pallas/Mosaic kernels** — not CPU stand-ins. This is a deliberate reversal of the prior CPU-first plan, driven by hard evidence from the prior **DeepSeek-V4** bring-up.

**The evidence (from `/home/enyouki/claude-deepseek-v4`):** that project was a multi-month chronicle of **TPU-only bugs that CPU structurally could not reproduce**. The dominant, most expensive class was **uninitialized-HBM-on-reshard ("S1")**: a device-side weight reshard read uninitialized HBM and baked per-process garbage into the weights, producing **coherent-looking-but-wrong** output at temp=0. It took ~30 debugging sessions and a fully-overturned root-cause hypothesis. Siblings: empty/idle-shard garbage matmuls (`gmm_v2(zero_initialize=True)` fix), `max-num-seqs>1` silent MoE corruption, Mosaic kernels needing `shard_map` wrapping, `E2003` tile-alignment failures, multi-host SPMD launch races, and fp4→fp8 casts that were bit-exact on CPU but diverged by 5.2e5 on v6e.

**The sharpening insight:** these expensive bugs are **multi-device + real-reshard-path** bugs. They do **not** appear on single-device, and many don't appear under the CPU `interpret=True` kernel path. So "develop on TPU" must specifically mean **exercise the real kernels AND the real multi-device sharded weight-loader early** — not merely "run tiny tests on one TPU chip" (that only buys the Mosaic-lowering / tile-alignment class).

**What we keep from CPU-first (do NOT discard):** the fp32 math oracle. We relocate it onto the TPU. A **pure-`jnp` fp32 reference runs on the TPU alongside the real kernel** and is the kernel's *answer key*. This is **not** a CPU replica — it is what makes a TPU failure debuggable: it turns an unattributable 4-way differential (math vs kernel vs sharding vs runtime) into single-delta comparisons, and preserves a tight fp32 math gate so a real ~1% math bug cannot hide under bf16 kernel noise. (See §7.)

**Three load-bearing guardrails the plan must keep (from adversarial review):**
1. A pure-`jnp` fp32 reference **runnable on TPU** — the irreplaceable middle oracle.
2. **Staged triangulation** with a tight fp32 *math* gate distinct from a loose bf16 *kernel-noise* gate. Never collapse "kernel-on-TPU vs HF-eager" into one number.
3. **Multi-device** on the real v6e is a **required** gate, not optional. Single-device v6e ≠ TPU validation for the bug class that motivated this pivot.

## 1. Goal & non-goals

### Goal
Bring up **GLM 5.2** — architecture `GlmMoeDsaForCausalLM` (`model_type="glm_moe_dsa"`), i.e. **DeepSeek-V3.2-style MLA + DeepSeek Sparse Attention (DSA) at GLM-5 scale** — as a **native-JAX** model in `tpu-inference`, **developed and validated on TPU with the real kernels from the start**. **DSA is implemented and validated, not deferred.**

The v1 finish line is **single-sequence correctness of the components on a smaller (scaled-down) config, on TPU, single- and multi-device** — the dense MLA backbone, the MoE/router stack, and the DSA sparse path — validated against the in-tree `transformers GlmMoeDsaForCausalLM` eager forward via the triangulation in §7. **We do not run the full 78-layer model end-to-end in v1** (that, and real-checkpoint generation, are Phase 3+).

### Hardware & environment reality (prerequisite — see Phase 0)
- Host is **worker of a v6e-16 pod** (machine type `ct6e-standard-4t-tpu`): **4 Trillium chips/host** via `/dev/vfio/{0,1,2,3}`; a single-host **4-chip mesh** is the multi-device test surface; the full 16-chip pod needs multi-host (Ray/Pathways) and is a later step.
- **The TPU is currently 100% occupied** by a running `gpt-oss-120b` vLLM job holding all 4 local chips. **Phase 0 must free a device** (stop that job, or use a free worker in the pod) before any kernel iteration.
- **Env:** system Python is 3.10 and **cannot** install the pinned `jax==0.10.1` (needs ≥3.11; repo Docker uses 3.12). There is no ready venv — Phase 0 stands one up. The persistent XLA **compilation cache is on by default** (`compilation_manager.py`), so edit→run is near-instant after the first cold compile.

### 1.1 What changed vs the (architecture-wrong) superseded spec — keep this
The original spec targeted `Glm4MoeLiteForCausalLM` = **GLM-4.7-Flash**, not GLM 5.2. Corrected here:
- **Class/identity:** `GlmMoeDsaForCausalLM` / `glm_moe_dsa`, config `GlmMoeDsaConfig(Glm4MoeLiteConfig)`. The Lite class has **no indexer at all**.
- **Scale:** GLM-5 (hidden **6144**, **78** layers, **256** experts, q_lora **2048**, **64** heads), not Lite ~30B (2048/47/64).
- **RoPE:** **non-interleaved (NeoX / rotate-half)** for *both* main MLA and the indexer. DSA removes Lite's `rope_interleave` (it raises `AttributeError`). ⚠️ The DSA module docstrings *claim* "interleaved"; the **code is non-interleaved** — trust the code.
- **Indexer ReLU:** the indexer **applies `F.relu`** to the scores (`modeling_glm_moe_dsa.py:219`).
- **`routed_scaling_factor`:** default **2.5** (config-driven; real checkpoint authoritative).
- **`num_experts_per_tok`:** **8**.
- **Layer schedule fields:** **no `scoring_func` / `first_k_dense_replace` field** on `GlmMoeDsaConfig`. Dense-vs-MoE from `mlp_layer_types` (HF) / `first_k_dense_replace` (vLLM/checkpoint); indexer recompute-vs-reuse from `indexer_types`. Scoring is hardcoded **sigmoid**.
- **Oracle:** the in-tree `transformers GlmMoeDsaForCausalLM` eager class (CPU, random weights, no download), not hand-transcribed numpy refs (those were tautological — they re-derived the code's own math).

### Non-goals (explicitly deferred past v1)
- **Full end-to-end model** (all 78 layers, real checkpoint generation). v1 validates components on a scaled-down config; Phase 3 begins real-checkpoint work.
- **MTP** (multi-token prediction). The checkpoint's extra `model.layers.78` MTP head is **dropped on load** (`_keys_to_ignore_on_load_unexpected = [r"model\.layers\.78.*"]`).
- **Quantization** (fp8 UE8M0 / fp4). v1 runs bf16/fp32. The GPU DSA path is fp8 (DeepGEMM `fp8_mqa_logits`); v1 uses a bf16/fp32 indexer.
- **Aggressive kernel autotuning.** Kernels are written **idiomatic and performant from the start** and reuse existing tuned params, but *autotuning* is a **bounded, post-correctness** activity (§11), not a continuous loop during bring-up.
- **Multi-batch / concurrent decode.** v1 is single-sequence. (The fork has a **confirmed** `max_num_seqs>1` MoE-corruption bug — same uninit-HBM mechanism; out of v1 scope.)

## 2. Architecture summary

GLM 5.2 is a decoder-only transformer where every layer is **MLA + DSA**, the FFN is **MoE** (after a few leading dense layers):

1. **MLA (Multi-head Latent Attention)** — cache one small **latent** (`kv_lora_rank=512` + `qk_rope_head_dim=64` shared rope key = 576 numbers) and **up-project** (`kv_b_proj`) to per-head K/V at attention time. Query path is compressed (`q_a_proj → q_lora_rank=2048 → q_a_layernorm → q_b_proj`). RoPE (non-interleaved) applies only to the 64-dim rope slice; the attention splits **nope-first** `[qk_nope=192, qk_rope=64]`. Softmax scale = `qk_head_dim**-0.5 = 256**-0.5` (**no mscale** under default rope; see §4.9). **No per-head QK-norm** — only `q_a_layernorm`(RMSNorm 2048) and `kv_a_layernorm`(RMSNorm 512).
2. **DSA (lightning indexer)** — a cheap auxiliary scorer ranks all past tokens and selects the **top-`index_topk`=2048**; MLA attention then runs only over the selected set (an additive `-inf` mask combined with the causal mask). V3.2/GLM form: per-head dot-product, **`relu`** on the scores, weighted head-sum, **no softmax**; `k_norm` = **LayerNorm** (eps 1e-6, the one LayerNorm in an otherwise RMSNorm model). The indexer splits **rope-first** `[qk_rope=64, rest=64]` (opposite order to main attention) and **reuses the MLA query LoRA residual** `q_a_layernorm(q_a_proj(x))`. Some layers (`indexer_types[i]=="shared"`) **reuse the previous layer's top-k**.
3. **MoE** — 256 expert FFNs (`moe_intermediate_size=2048`) + 1 always-on shared expert; a router picks **top-8-of-256**. **sigmoid** scoring; `e_score_correction_bias` added **only for selection** (gathered weights use the bias-free sigmoid scores); `norm_topk_prob` renormalizes; routed-expert weights scaled by `routed_scaling_factor` (default 2.5); the shared expert added **unscaled**. The first `first_k_dense_replace` (default 3) layers are a plain dense SwiGLU FFN (`intermediate_size=12288`).

**Key safety property (drives the validation ladder):** for any sequence of length ≤ `index_topk` (2048), `topk = min(index_topk, total_len)` selects **every** valid token, so DSA and dense attention produce the **identical** result. Sparsity only changes behavior past 2048 tokens. This lets us validate a correct **dense backbone** first (short sequences) and then the **sparse path** (long sequences) with a clean equivalence check at the boundary — and crucially, that equivalence is checkable **on TPU at fp32** (≈1e-6), isolating the entire sparse-plumbing bug class with no HF dependency.

## 3. Exact GLM 5.2 config (`GlmMoeDsaConfig`)

`GlmMoeDsaConfig` subclasses `Glm4MoeLiteConfig` (`modular_glm_moe_dsa.py:81`); the generated config flattens inherited fields. **Defaults below were verified live** via `GlmMoeDsaConfig()`. The real checkpoint's `config.json` is authoritative and overrides these; our config path reads everything from `hf_config`.

| Field | Default | Field | Default |
|---|---|---|---|
| `hidden_size` | 6144 | `n_routed_experts` | 256 |
| `num_hidden_layers` | 78 | `n_shared_experts` | 1 |
| `intermediate_size` (dense FFN) | 12288 | `num_experts_per_tok` | **8** |
| `moe_intermediate_size` | 2048 | `n_group` / `topk_group` | **1 / 1 (grouping OFF)** |
| `vocab_size` | 154880 | `routed_scaling_factor` | **2.5** |
| `max_position_embeddings` | 202752 | `norm_topk_prob` | True |
| `num_attention_heads` | 64 | scoring | **sigmoid (hardcoded; no `scoring_func` field)** |
| `num_key_value_heads` | 64 | `rms_norm_eps` | 1e-5 |
| `q_lora_rank` | 2048 | `attention_bias` | False |
| `kv_lora_rank` | 512 | `tie_word_embeddings` | False |
| `qk_nope_head_dim` | 192 | `qk_head_dim` (derived) | 256 |
| `qk_rope_head_dim` | 64 (`head_dim` attr → this) | `initializer_range` | 0.02 |
| `v_head_dim` | 256 | `hidden_act` | "silu" |

**RoPE config key is `rope_parameters`** (a dict), **not `rope_scaling`**. Default `rope_parameters = {'rope_theta': 10000.0, 'rope_type': 'default'}` — **plain RoPE, no YaRN** by default. A real GLM-5 checkpoint may set `rope_type="deepseek_yarn"` with `factor`/`mscale_all_dim`; see §4.9 for the mscale consequence.

**Layer schedules** (two independent per-layer patterns, computed in `__post_init__`):
- **Dense-FFN vs MoE → `mlp_layer_types`**: if absent, `["dense"]*min(3,L) + ["sparse"]*(L-3)` ⇒ **first 3 layers dense**, rest MoE. vLLM reads the equivalent `config.first_k_dense_replace`. **Our config path reads whichever the checkpoint supplies** (prefer `first_k_dense_replace` if present, else derive from `mlp_layer_types`).
- **Indexer recompute vs reuse → `indexer_types`**: if absent, computed from `index_topk_freq` — a `"full"` layer at the cadence start, the rest `"shared"`; **not** uniformly `"full"`. A `"shared"` layer reuses the previous layer's `topk_indices` (`skip_topk`). **Every attention layer is DSA-sparse**; "full"/"shared" only controls whether the *indexer scores* are recomputed. **Read `indexer_types` from `hf_config`**; the scaled-down test config sets it explicitly with ≥1 `"shared"` layer to exercise the reuse path.

**DSA indexer fields:** `index_topk=2048`, `index_head_dim=128`, `index_n_heads=32`, `indexer_types` (computed). All read from `hf_config`; never hardcoded.

**Sentinels (DSA disables these — they raise `AttributeError`):** `rope_interleave`, `pretraining_tp`. Never reference them.

**Norms — definitive:**
- `q_a_layernorm` = RMSNorm(`q_lora_rank=2048`, eps `rms_norm_eps`)
- `kv_a_layernorm` = RMSNorm(`kv_lora_rank=512`, eps `rms_norm_eps`)
- `input_layernorm`, `post_attention_layernorm`, final `model.norm` = RMSNorm(`hidden_size`)
- (indexer) `k_norm` = **LayerNorm**(`index_head_dim=128`, eps **1e-6**) — has **weight AND bias**. The only LayerNorm in the model.
- **No per-head q/k norm.**

## 4. GLM/DSA deltas vs the existing DeepSeek-V3 JAX path (the things to get exactly right)

The repo's `models/jax/deepseek_v3.py` is **plain DeepSeek-V3 (no indexer)**; its module globals are already close to GLM (`routed_scaling_factor=2.5`, `num_experts_per_token=8`, `first_k_dense_replace=3`, 256 experts). The deltas below are each a known foot-gun.

1. **Config-driven hparams** — replace DeepSeek's hardcoded module globals (`deepseek_v3.py:76-108`) with values from `hf_config` (à la `qwen3_moe.py`). GLM differs in: `n_group` (1 vs 8), `hidden_size` (6144 vs 7168), `q_lora_rank` (2048 vs 1536), `qk_nope_head_dim` (192 vs 128), `v_head_dim` (256 vs 128), `num_attention_heads` (64 vs 128), and adds the `index_*` fields.
2. **`n_group = topk_group = 1`** — grouping **off**; take top-8 from all 256. `DeepSeekV3Router.get_topk_indices` already skips the group branch when `n_groups==1` (`deepseek_v3.py:1057`).
3. **Router gate in fp32**, **sigmoid** scoring, `e_score_correction_bias` added **before top-k for selection only**; the **gathered weights use bias-free sigmoid scores**; `norm_topk_prob` renorm (`/ (sum + 1e-20)`).
4. **`routed_scaling_factor` (default 2.5) applied to the routed-expert output**, **not** to router logits; shared expert added **unscaled**. Site already correct in `SharedFusedMoe.__call__:830`; pass the config value through `DeepseekV2Moe`.
5. **Non-interleaved (NeoX / rotate-half) RoPE for BOTH** main MLA and the indexer. **The single highest porting risk** (the fork uses complex-multiply RoPE; DeepSeek-V3's `DeepseekScalingRotaryEmbedding` is interleaved). Pin it bit-for-bit vs HF `apply_rotary_pos_emb` (§7 rec C). Main attention splits **nope-first** `[192,64]`; indexer splits **rope-first** `[64,64]`.
6. **silu only** for FFN/expert activation (gated SwiGLU).
7. **Dense-then-MoE schedule** — on `GlmMoeDsaConfig` the controlling field is `mlp_layer_types` (default first-3-dense); there is **no `first_k_dense_replace` field** on the HF config (it's the vLLM/checkpoint key). Config-driven; read whichever the checkpoint supplies (§3).
8. **No per-head QK-norm** (§3).
9. **DSA softmax scale = `qk_head_dim**-0.5 = 256**-0.5` with NO mscale** under default rope. ⚠️ **HF-vs-vLLM discrepancy:** under a YaRN checkpoint, **vLLM** multiplies the scale by `mscale²` (`deepseek_v2.py:515`), but **HF DSA has no mscale branch**. With default rope both agree. v1 tests use **default rope** (no ambiguity). The YaRN `mscale²` is a **Phase-3 reconciliation risk** requiring the real `config.json` (§9, §10).
10. **DSA indexer = V3.2/GLM form, NOT the fork's V4 form.** Keep: q-up-proj from the shared q-LoRA residual, **`relu` on scores**, per-head `weights_proj × n_heads**-0.5`, head-sum, `top_k`. **Drop** (V4-only): the **compressor/CSA** (GLM's K is direct `k_norm(wk(hidden_states))` per-token — the single biggest math change), SWA flavors, mHC sinkhorn, APE. `k_norm` is **LayerNorm**, not RMSNorm.
11. **PyTorch quirks — nothing to port:** the layer-0 residual `.clone()` and the fp16-overflow rescale are **DeepSeek-V3 lineage quirks the HF GLM-DSA port already dropped**. Don't carry them over if copying DeepSeek-V3 code as a starting point.

## 5. Code structure (TPU-forward)

**New file `tpu_inference/models/jax/glm_moe_dsa.py`, reusing DeepSeek modules by import.**

> **TPU-forward note:** Three things from the CPU-first rework are now done differently because everything runs on TPU. (a) We implement **two** MLA paths from the start and run **both on TPU**: a **pure-`jnp` non-absorbed `GlmMoeDsaAttention`** (the fp32 *oracle*) and the **production absorbed MLA Pallas kernel** (`kernels/mla/v2`). The jnp-ref is the kernel's answer key (§7), not a CPU crutch. (b) **RoPE has no flag**: the repo's `DeepseekScalingRotaryEmbedding` is hard-coded *interleaved*; GLM is rotate-half — and **the base `RotaryEmbedding` in `rope.py` is already rotate-half**, so subclass/reuse the base; do **not** use `DeepseekScalingRotaryEmbedding`. v1 uses default rope (no YaRN). (c) The production **`JaxAutoWeightsLoader` regex loader** is brought up **early** (not deferred), because the most expensive prior bug (S1, uninit-HBM-on-reshard) lives in the **multi-device sharded weight-load path** — the multi-device gate (§6, §7) needs it.

- **Import & reuse** from `deepseek_v3.py`: `DeepseekV2Moe`, `DeepSeekV3Router`, `DeepseekV3MLP`, `SharedFusedMoe`/`JaxMoE`, the `MLAEinsum` split math. Use `JaxRmsNorm`/`JaxLayerNorm` (`norm.py`), `JaxEinsum`/`JaxLmHead` (`linear.py`), `JaxEmbed` (`embed.py`), `UnquantizedConfig({})` (`quantization/unquantized.py`). For the **absorbed MLA kernel path**, supply a **no-quant `kv_b` split** producing `k_up_proj`/`v_up_proj` (these are created during weight load, not random-init — `MLAEinsum.load_weights` asserts `quant_config is not None`, so write a standalone no-quant split rather than touching it).
- **Write GLM-specific deltas only:**
  1. **Config-driven hparams** read from `hf_config`, replacing DeepSeek's module globals.
  2. **DSA `GlmMoeDsaIndexer` module + top-k selection** (the genuinely new piece; V3.2/GLM form per §3/§4).
  3. **Non-interleaved RoPE** wiring + GLM router constants.
  4. **DSA sparse-attention kernel** — adapt the fork's Mosaic `sparse_attn` kernel (Phase 2): drop `attn_sink`, drop the compressor (direct `k_norm(wk(x))`), one-hot-matmul gather for tile alignment (the fork hit `E2003 UnprovenMemoryAccessAlignment` on v6e and fixed it this way), `-1` sentinel = ignore. The fork's kernel is **not-yet-wired and non-paged** — wire it and give it the paged/flattened KV surface.
- **Surgical, behavior-preserving param-izing of DeepSeek modules:** `DeepseekV2Moe.__init__` (and the `SharedFusedMoe(...)` block, incl. the `expert_axis_name` global — `deepseek_v3.py:908`) read ~10 module globals. Add optional kwargs defaulting to current globals so existing DeepSeek behavior is **unchanged** (existing DeepSeek tests stay green as a gate).
- **DSA hook point:** the indexer + additive `-inf` mask insert into the new `GlmMoeDsaAttention.__call__` for the jnp-ref path (a `dsa` flag toggles dense vs sparse); on the kernel path the hook is `DeepseekV3MLA.compute_attention` (`deepseek_v3.py:663`) immediately before `mla_attention(...)` (`:702`), threading `topk_indices` into the kernel.
- **New runtime scaffolding for DSA (Phase 2):** an **indexer-key KV cache** (raw per-token keys `[tokens, index_head_dim=128]`, single-head/MQA, distinct from the MLA latent cache); a **`topk_indices` field added to `AttentionMetadata`** (`attention_metadata.py`), `int32`, shape `[tokens, index_topk]`, `-1` sentinel; runner-side population; threading runner → `mla_attention` → kernel scalar-prefetch.
- **Registration:** `_MODEL_REGISTRY["GlmMoeDsaForCausalLM"] = GlmMoeDsaForCausalLM` in `models/common/model_loader.py` (inline-import pattern `:73-100`); add the arch to `_PP_DISABLED_MODELS` (`:60-61`). If a checkpoint declares a custom `auto_map` arch string, map it to the same class.

**Weight names** (HF checkpoint layout, prefix `model.layers.{i}.`) — our loader maps the **unfused HF names** (vLLM's `fused_qkv_a_proj`/`wk_weights_proj` are internal post-load fusions, NOT on disk):
- **Attention:** `self_attn.{q_a_proj, q_a_layernorm, q_b_proj, kv_a_proj_with_mqa, kv_a_layernorm, kv_b_proj, o_proj}.weight` — `kv_a_proj_with_mqa` concatenates `[kv_lora_rank(512) | qk_rope_head_dim(64)]`; split `kv_b_proj`→`k_up`/`v_up`.
- **Indexer:** `self_attn.indexer.{wq_b, wk, weights_proj}.weight` + `self_attn.indexer.k_norm.{weight, bias}` (LayerNorm ⇒ has bias).
- **MoE (sparse layers):** `mlp.gate.weight` + `mlp.gate.e_score_correction_bias`; **fused** experts `mlp.experts.gate_up_proj` (3-D `[E, 2·moe_inter, hidden]`, **split gate/up**) and `mlp.experts.down_proj` (3-D `[E, hidden, moe_inter]`); `mlp.shared_experts.{gate_proj, up_proj, down_proj}.weight` (**un**fused).
- **Dense layers (0..first_k_dense_replace-1):** `mlp.{gate_proj, up_proj, down_proj}.weight` (un-fused).
- **Top-level:** `model.embed_tokens.weight`, `model.norm.weight`, `lm_head.weight` (**untied**). Drop `model.layers.78.*` (MTP).

## 6. Phase plan (TPU-first)

### Phase 0 — TPU env, HF oracle, harness
**Deliverable:** a working TPU JAX environment and a runnable harness whose oracle is the in-tree `transformers GlmMoeDsaForCausalLM` eager forward (torch, CPU).
- **Free a TPU device** (stop the `gpt-oss-120b` job or use a free pod worker) and stand up a **Python 3.12 venv**: `jax[tpu]==0.10.1`, `libtpu`, `flax`, `transformers` (≥5.9.0), `torch`, `numpy`, `pytest`, `pytest-mock`. Verify `jax.devices()` shows the v6e chips. Compilation cache is on by default.
- **HF-eager oracle fixture** (torch CPU, random weights, no download): `GlmMoeDsaForCausalLM.from_config(cfg, attn_implementation="eager")`, **`experts_implementation="eager"`** (force the genuine per-expert loop, not the fused `grouped_mm`), deterministic init **including buffers** (`e_score_correction_bias` — else the bias-for-selection delta passes vacuously).
- **Tiny `GlmMoeDsaConfig`** — min-viable dims that still exercise the real kernels: `hidden=512`, `layers=4` (≥1 dense + ≥1 MoE + ≥1 indexer "full" + ≥1 "shared"), `heads=8`, `index_topk=64`, `n_routed_experts=8`, `num_experts_per_tok=2`, `page_size=16`, **default rope**. Keep `kv_lora_rank=512 / qk_rope=64 / qk_nope=192 / v_head_dim=256 / index_head_dim=128` at **real** values (they auto-pad identically and cost ~nothing). Use two seq regimes: ~32 (≤topk, dense-equiv) and ~128 (>topk=64, true sparsity).
- **Harness helpers:** a `t2j` weight converter (**transpose every linear** — JAX `x@kernel (in,out)` vs HF `nn.Linear (out,in)`; `embed_tokens` is the exception, `lm_head` transposes; split fused `gate_up_proj`), a `maxabs` comparison helper (upcast to fp32 **inside** maxabs), and an identical-weights checksum (sum of copied params both sides) asserted before every parity run.
- **Mesh fixtures:** the existing `conftest.py` `mesh` fixture asserts `num_devices==1`. Provide **both** a 1-device fixture and a **multi-device (4-chip) mesh** fixture (do not inherit the single-device assert). Do **not** depend on `test_deepseek_v3.py` (broken import `DeepSeekV3WeightLoader`; CI ignores it) — use construction tests for the DeepSeek regression gate.
- **Exit criterion:** `jax.devices()` sees the v6e; the harness collects on TPU; the HF oracle runs a finite forward on the tiny config; a trivial JAX op runs on TPU and round-trips to host numpy.

### Phase 1 — Dense-MLA GLM backbone on TPU (real kernel + fp32 jnp-ref oracle)
**Deliverable:** a complete, registered, config-driven `glm_moe_dsa.py` whose **dense backbone** (MLA + MoE + router + RoPE; indexer present but trivially selecting all tokens on short seqs) matches HF eager **on TPU**, with the **real absorbed MLA kernel** validated against the fp32 jnp-ref. This is the gate that catches the S1/uninit-HBM/reshard class.
- **Exit criterion:** full backbone forward on the tiny config, seq < `index_topk` — fp32 **math gate** at rtol≈1e-3 vs HF; bf16 **shipped gate** within empirical kernel tolerance (~5e-2…2e-1) + argmax ≥0.95; **multi-device == single-device** at fp32; weight-map golden passes.

## 7. Testing & validation strategy

**Oracle = the in-tree `transformers GlmMoeDsaForCausalLM` eager forward** (torch, CPU, random weights, no download) — an **independent** implementation of every §4 delta. It runs on the host; the model-under-test runs on TPU; comparison is a **host-side `maxabs`** (JAX arrays → host numpy via `np.asarray`/`device_get`, device-agnostic). Upcast both sides to fp32 **inside** `maxabs` (a bf16 TPU output returns as numpy bf16 — this is load-bearing, not cosmetic). No `jax_enable_x64`.

### The triangulation (the core of the rework)
Three implementations, three single-delta comparisons — never a fused HF-vs-bf16-kernel verdict:

```
HF-eager (CPU, fp32)  ──[MATH gate, 1e-3]──►  jnp-ref (TPU, fp32)  ──[KERNEL gate]──►  Pallas kernel (TPU)
        independent oracle                      our answer key                          shipped path
```

- **MATH gate** (HF-fp32 vs jnp-ref-fp32-on-TPU, rtol≈1e-3): catches every §4 math/spec bug (RoPE convention, head splits, `256**-0.5` scale, kv_b split, router bias-for-selection, routed_scaling 2.5, relu placement). This is the gate with *teeth* — prove it by an **injected-1%-error test** (deliberately perturb a scale and assert the fp32 gate catches it). Without an fp32 reference on TPU, there is **no** math gate, because the bf16 kernel noise floor (~4e-3 per element, compounding over depth) is larger than a real ~1% bug.
- **KERNEL gate** (jnp-ref vs kernel, on TPU): run the kernel **twice** — (a) fp32 (`s_dtype=fp32, p_same_dtype_as_v=False`) vs jnp-ref-fp32 at tight tolerance to check kernel *algebra*; (b) bf16 shipped defaults vs jnp-ref-bf16 at the **empirically-measured bf16 floor** (run the jnp-ref itself in bf16 to derive the floor — don't guess). Repo precedent for bf16 attention tolerances: ~0.1–0.2 (`ragged_paged_attention_kernel_v2_test.py:129-134`, `mla_v2_test.py:405`); MoE/elementwise ~5e-2.

### Tiered comparison (each submodule at the layer where it lives, plus full forward)
| Check | Tolerance / method |
|---|---|
| RoPE (rec C, bit-for-bit) | exact / 1e-6 |
| Router | top-k indices **exact** (fp32 gate, well-separated scores) + weights 1e-2 |
| RMSNorm / LayerNorm, dense FFN, embed/lm_head | 1e-3 (fp32 math gate) |
| MLA attention | math gate 1e-3 (jnp-ref vs HF); kernel-algebra 1e-3 fp32; shipped bf16 at measured floor |
| MoE experts (vs **eager** `GlmMoeDsaNaiveMoe` — assert `config._experts_implementation == "eager"`) | 1e-2 |
| Indexer | **boundary-aware tie-tolerant** top-k set equality + index_scores 1e-3 fp32 |
| Dense==sparse (§2, seq ≤ topk) | **fp32 ≈1e-6 on TPU** — isolates sparse plumbing, no HF dependency |
| Full forward | per-layer index-**set** equality (tie-tolerant) + fp32 math gate 1e-3 / bf16 shipped at floor + argmax ≥0.95 |

### Multi-device (required gate, not optional)
The bug class that motivated TPU-first lives here. On the **4-chip mesh**, via the **real sharded weight-loader**:
- **N-device fp32 result == 1-device fp32 result** (sharding must be value-invariant). Compare 1/2/4-device.
- **NaN-poison HBM** before weight load; assert no NaN/garbage leaks into outputs (catches uninit-HBM reads on reshard — the S1 mechanism). Watch for the empty/idle-shard garbage-matmul pattern (`gmm_v2(zero_initialize=True)`, output zeroing of non-finite values).
- Exercise `expert_axis_name` partitioning. **A single-device v6e pass is NOT TPU validation.**

### Determinism & ties
TPU is seeded-reproducible but **not bit-equal** to the torch ref; reductions are non-associative and reduction order differs (more so multi-device). Top-k ties are a documented hazard (`fused_moe_v1_test.py:299-300`, jax #34620). Hence: fp32 math gate uses well-separated scores where exact equality is asserted; the indexer set compare is **boundary-aware tie-tolerant** (adjudicate boundary swaps by recomputed fp32 scores), and `index_scores` are gated separately at 1e-3 so a relu/scale bug shows up in scores even when the set survives. Pin the torch ref: fixed seeds, single-process (`WORLD_SIZE=1`), fp32, eager.

### What this buys / doesn't
Triangulation on TPU kills the dominant new-model bug class (weight mapping/fusion, router logic, MLA/MoE math, RoPE/head-split/scale, indexer top-k) **and** the TPU-only classes the prior project paid for (Mosaic lowering, tile alignment, sharding/uninit-HBM reshard, paged-KV/metadata threading). It does **not** yet cover (Phase 3): the YaRN `mscale²` reconciliation against the real checkpoint, fp8/fp4 drift, the full 16-chip multi-host pod, the multi-batch MoE bug, or full 78-layer end-to-end behavior.

## 9. Risks & mitigations

| Risk | Mitigation |
|---|---|
| **Uninit-HBM / reshard / SPMD bugs** (S1 class) — silent, coherent-looking-but-wrong; CPU-invisible; cost ~30 sessions last time | **First-class multi-device gate from Phase 1** (§7): real sharded loader on 4-chip mesh, N-dev==1-dev at fp32, NaN-poison HBM, host-stack-into-sharded-layout, `gmm_v2(zero_initialize=True)`. |
| **Unattributable TPU failures** (math vs kernel vs sharding vs runtime) | **Triangulation** (§7): keep a pure-`jnp` fp32 reference runnable on TPU; every comparison is single-delta. |
| **bf16 tolerance inflation hides a real ~1% math bug** | **fp32 math gate** distinct from bf16 kernel-noise gate; derive the bf16 floor empirically; **injected-1%-error test** proves the fp32 gate has teeth; argmax is a backstop, not the math verdict. |
| **Top-k set equality flaky under bf16/multi-device reduction order** | **Boundary-aware tie-tolerant** set compare (adjudicate boundary swaps by fp32 scores); separate `index_scores` 1e-3 gate; fp32 exact-equality only on well-separated scores. |
| **YaRN `mscale²` HF-vs-vLLM divergence** (§4.9): HF DSA omits it; vLLM applies it under `deepseek_yarn` | v1 uses **default rope** (both agree). Phase-3 task in **fp32** against the real `config.json`; never judged only at bf16 (§10). |
| **RoPE convention** — GLM is rotate-half/NeoX; the fork is complex-multiply; `DeepseekScalingRotaryEmbedding` is interleaved | Reuse the base rotate-half `RotaryEmbedding`; **pin bit-for-bit vs HF** as its own test (rec C). |
| **Indexer ReLU / sparse-plumbing bugs** (gather off-by-one, `-1` sentinel, mask order) | **Apply relu** (match HF). **Dense==sparse fp32 equivalence at seq ≤ topk** catches the plumbing class exactly with no HF dependency. |
| **Weight fusion / transpose mismatch** | Loader maps **unfused HF names**, splits `gate_up_proj`; **weight-mapping golden test** + identical-weights checksum + per-submodule fp32 parity (don't rely on end-to-end). |
| Param-izing DeepSeek globals regresses DeepSeek (esp. `expert_axis_name`) | Optional args defaulting to current globals; **construction regression tests** (rec E); don't depend on broken `test_deepseek_v3.py`. |
| **Absorbed MLA needs `k_up`/`v_up`** (created on load, asserts `quant_config is not None`) | Standalone **no-quant `kv_b` split** producing `k_up`/`v_up`; leave `MLAEinsum.load_weights` untouched. |
| **Kernel-precision wrapper rigidity** — `mla_attention` hardcodes bf16 knobs | Thread `s_dtype`/`p_same_dtype_as_v`/`two_step_flash_attention` through the wrapper so fp32 algebra checks are possible. |
| **Device contention / env friction** — TPU busy with another job; py3.10 can't install jax; no venv | Phase 0 frees a device + stands up a py3.12 venv; compile cache reused across runs for fast iteration. |
| **Adapted sparse kernel** is not-yet-wired, non-paged, carries `attn_sink`, may hit tile-alignment | Treat as an **adapt** (Phase 2): drop `attn_sink`/compressor, one-hot gather, wire paged KV + `topk_indices`; validate kernel-vs-jnp-ref. |

## 11. Kernel optimization — idiomatic now, bounded tuning later

Performance is pursued in two distinct modes; neither is a continuous autonomous loop during correctness bring-up.

**Now (free, during bring-up):**
- Write kernels in the **performant TPU idiom from day 1** (sensible block/tile structure, avoid pathological layouts, `shard_map`-wrapped Mosaic, host-side RoPE-freq precompute).
- **Reuse existing tuned MLA params** via `get_tuned_params` (`tuned_params.py`) where shapes match.

**Later (bounded, post-correctness):** *autotuning* starts only **after single-sequence correctness is green and the kernel spec is frozen** (notably the YaRN `mscale²` question, §10). Then:
- Run **bounded, human-triggered sweeps** at **real GLM-5.2 dims** (perf on tiny configs is meaningless — the serving lookup is exact-shape-keyed, so tiny-shape tuning never hits production) using the existing tuner (`tools/kernel/tuner/v1`, `--run_locally=True`).
- For DSA, **write a dedicated `sparse_attn` tuner** (subclass `KernelTunerBase`) once the kernel is correct — the existing tuner is MLA-decode-only and has no feedback loop / apply-back.
- **Guardrails (mandatory):** a tuned config is accepted only if the **full correctness suite still passes** with it applied; a **tuned-vs-baseline regression test** (≤5% slower, modeled on `mla_tuned_vs_baseline_test.py:131-135`, extended to GLM/DSA shapes); tuning runs against a **frozen kernel commit**; apply-back to `tuned_params_mapping` is **human-reviewed**, never silently mutated by an agent.
- **Stop criterion:** discrete bounded sweeps, not a perpetual loop — stop when the best config is within timing noise (<3–5%) of the previous best, or at a fixed TPU-hour budget.
