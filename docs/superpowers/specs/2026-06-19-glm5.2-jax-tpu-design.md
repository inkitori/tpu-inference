## 1. Goal & non-goals

### Goal
Bring up **GLM 5.2** as a **native-JAX** model in `tpu-inference`, validated for **mathematical correctness on CPU first**, then on TPU. The v1 finish line is **single-sequence correctness** (CPU parity vs a HuggingFace-eager reference, then single-sequence correctness on TPU).

### Non-goals (explicitly deferred past v1)
- **MTP** (multi-token prediction / speculative decode) — `Glm4MoeLiteMTP`. Cleanly skippable; not built in v1.
- **Quantization** (fp8 / fp4 weights & activations). v1 runs bf16/fp32.
- **Performance work** — kernel tuning, throughput. v1 only needs *correct*, not fast.
- **Multi-batch / concurrent decode.** v1 is single-sequence. (The `../claude-deepseek-v4` fork has a known multi-batch MoE corruption bug; solving it is a Phase 3 concern, not v1.)

These are listed so the implementation plan does not silently pull them in.

## 2. Architecture summary

GLM 5.2 is **DeepSeek-V3-shaped + DeepSeek Sparse Attention (DSA)**. A decoder-only transformer where both halves of each layer are the cheaper variants:

1. **MLA (Multi-head Latent Attention)** — instead of caching full per-head K/V (~10,240 numbers/token/layer), cache one small **latent** (`kv_lora_rank=512` + `qk_rope_head_dim=64` shared rope key = 576 numbers, ~18× smaller) and **up-project** (`kv_b_proj`) to per-head K/V at attention time. Query path is likewise compressed (`q_a_proj → q_lora_rank=768 → q_a_layernorm → q_b_proj`). RoPE applies **only** to the 64-dim rope slice. **No per-head QK-norm** (see §4).
2. **DSA (lightning indexer)** — a cheap auxiliary scorer ranks all past tokens and selects the **top-2048**; expensive MLA attention runs only over those. V3.2 form: **raw weighted dot-product, no softmax, no ReLU**, `k_norm` = LayerNorm. Per-layer schedule keeps the **first ~2 layers dense**, the rest sparse.
3. **MoE** — 64 expert FFNs (`moe_intermediate_size=1536`) + 1 always-on shared expert; a router picks **top-4-of-64**. **sigmoid** scoring, `e_score_correction_bias`, `norm_topk_prob`, output scaled by `routed_scaling_factor=1.8`. The first `first_k_dense_replace` layers are a normal dense FFN (`intermediate_size=10240`).

**Key safety property (drives the phase plan):** for any sequence of length ≤ 2048, DSA and dense attention produce the *identical* result (top-2048-of-≤2048 selects everything). Sparsity only changes behavior past 2048 tokens. This lets us build and validate a correct dense GLM first (Phase 1), then add DSA (Phase 2) with a clean equivalence check.

## 3. Exact GLM 5.2 config

Target architecture is the GLM4-MoE **Lite** variant (`Glm4MoeLiteForCausalLM`, `model_type="glm4_moe_lite"`) — the one whose attention is **MLA + DSA** (delegates to DeepSeek's MLA attention + an `Indexer`). NOT plain `glm4_moe.py` (that is the dense-attention GLM-4.5/4.6/4.7 line, no MLA/DSA).

vLLM has no literal "GLM-5"/"5.2" arch string; GLM 5.2 reuses the Lite classes and supplies real hyperparameters from the **checkpoint's `config.json`**. DSA is gated solely by the presence of `index_topk` in the config.

**Hyperparameters** (Lite transformers defaults ≈ a 30B "GLM-4.7-Flash"; the real checkpoint overrides these. Used here as the reference shape; scaled *down* for CPU tests):

| Field | Value | Field | Value |
|---|---|---|---|
| `hidden_size` | 2048 | `n_routed_experts` | 64 |
| `num_hidden_layers` | 47 | `n_shared_experts` | 1 |
| `intermediate_size` (dense FFN) | 10240 | `num_experts_per_tok` | 4 |
| `moe_intermediate_size` | 1536 | `n_group` / `topk_group` | **1 / 1 (grouping OFF)** |
| `vocab_size` | 154880 | `routed_scaling_factor` | **1.8** |
| `max_position_embeddings` | 202752 | `norm_topk_prob` | True |
| `num_attention_heads` | 20 | `scoring_func` | "sigmoid" (hardcoded) |
| `num_key_value_heads` | 20 | `rms_norm_eps` | 1e-5 |
| `q_lora_rank` | 768 | `attention_bias` | False |
| `kv_lora_rank` | 512 | `tie_word_embeddings` | False |
| `qk_nope_head_dim` | 192 | `rope_interleave` | True |
| `qk_rope_head_dim` | 64 (`head_dim`→rope via attribute_map) | RoPE | `rope_parameters` (YaRN factor + `mscale_all_dim` → `scale *= mscale²`) |
| `v_head_dim` | 256 | | |

- `first_k_dense_replace` is **not** in the Lite HF dataclass (HF uses `mlp_layer_types`), but **vLLM reads `config.first_k_dense_replace`** — the checkpoint must supply it. Our config path must read it from `hf_config`.

**DSA indexer** (`Indexer`): enabled by `index_topk` (= 2048). `n_head = index_n_heads`, `head_dim = index_head_dim`, indexer rope dim = `qk_rope_head_dim` (64) — **all read from the checkpoint config**. (Code comments default to 64 heads / 128 head_dim; the architecture research note observed real GLM-5 may use 32 indexer heads. Because these are config-driven, we read the checkpoint value rather than hardcode.) Score math: **no ReLU, no softmax**; `weights = weights_proj_out · q_scale · softmax_scale · n_head^-0.5`, `softmax_scale = head_dim^-0.5`. GPU uses fp8 per-token-group quant; our JAX path uses bf16/fp32.

- **Per-layer sparsity schedule:** `index_topk_freq` (default 1), `index_topk_pattern` (default None → offset rule), `index_skip_topk_offset` (default 2). A layer is sparse iff `pattern[layer]=="S"`, else `max(layer - offset + 1, 0) % freq == 0`. With defaults ⇒ first ~2 layers dense, rest sparse.

**QK-norm — DEFINITIVE: there is NO per-head q/k norm in the MLA/Lite variant.** Only:
- `q_a_layernorm` = RMSNorm(`q_lora_rank`, eps=`rms_norm_eps`)
- `kv_a_layernorm` = RMSNorm(`kv_lora_rank`)
- (indexer) `k_norm` = **LayerNorm**(`index_head_dim`, eps **1e-6**) — separate, and the one LayerNorm in an otherwise RMSNorm model.

(Per-head `use_qk_norm` exists only in the *dense* `Glm4MoeAttention`, which we are not using.)

## 4. GLM deltas vs DeepSeek-V3 (the things to get exactly right)

These are the only behavioral differences from the existing DeepSeek-V3 implementation. Each is a known foot-gun:

1. **`routed_scaling_factor = 1.8`** (DeepSeek-V3 = 2.5), applied to the expert **OUTPUT** (`apply_routed_scale_to_output=True`), **not** to router logits.
2. **`n_group = topk_group = 1`** — grouping effectively **off** (DeepSeek-V3 = 8/4). Simpler: take top-4 from all 64; skip group-limited routing.
3. **Router gate in fp32**, **sigmoid** scoring + `e_score_correction_bias` (load-balancing bias added before top-k).
4. **`rope_interleave = True`** (GPT-J/interleaved style for MLA) + a **separate indexer rope** (NeoX style). Don't share rope style across the two.
5. **silu only** for expert/FFN activation (hard error otherwise).
6. **`first_k_dense_replace`** leading layers are dense FFN; the rest are MoE — config-driven.
7. **No per-head QK-norm** (see §3). Only the two LoRA-rank RMSNorms + indexer LayerNorm.
8. Reference-only PyTorch quirks to **skip** in JAX: residual `.clone()` on the first layer, fp16-overflow rescale (we run bf16/fp32).

**DSA indexer is the V3.2 form, NOT the fork's V4 form:** the `../claude-deepseek-v4` fork applies `relu(...)` in the indexer score (`deepseek_v4_attention.py:478-485`) and uses a compressor/CSA. **GLM does neither** — strip the ReLU and the compressor when adapting the fork. `k_norm` is LayerNorm, not RMSNorm.

## 5. Code structure (approved decision)

**New file `tpu_inference/models/jax/glm4_moe_lite.py`, reusing DeepSeek modules by import.**

- **Import & reuse** from `deepseek_v3.py`: the MLA attention stack, `DeepseekV2Moe`, `DeepSeekV3Router`, `DeepseekV3MLP`, `SharedFusedMoe`/`JaxMoE`, `DeepseekScalingRotaryEmbedding`, the `MLAEinsum` weight transform, the decoder layer, and the weight loader.
- **Write GLM-specific deltas only:**
  1. **Config-driven hparams** read from `hf_config` (à la `qwen3_moe.py:68,104-124`), replacing DeepSeek's hardcoded module globals (`deepseek_v3.py:76-108`).
  2. **DSA `Indexer` module + top-k selection** (the genuinely new piece; V3.2 form per §3/§4).
  3. **GLM router constants** — `routed_scaling_factor=1.8` on output, `n_group=topk_group=1`, plus the other §4 deltas.
- **Surgical param-izing of DeepSeek modules:** where a reused DeepSeek module currently reads a hardcoded global, parameterize it to accept the value from config. Keep changes minimal and **behavior-preserving for the existing DeepSeek path** (existing DeepSeek tests must still pass). Prefer adding an optional constructor arg defaulting to current behavior over rewriting.
- **DSA hook point:** the indexer + selection insert in `compute_attention` (DeepSeek `deepseek_v3.py:663`), immediately before the `mla_attention(...)` call (`:702`). In Phase 1 this hook is absent (pure dense MLA); Phase 2 adds it.
- **Registration:** add `"Glm4MoeLiteForCausalLM": Glm4MoeLiteForCausalLM` to `_MODEL_REGISTRY` in `models/common/model_loader.py`, and add the arch string to `_PP_DISABLED_MODELS` (pipeline parallelism not wired). If the real checkpoint declares a distinct arch string, map it to the same class.

**Weight names** (HF, prefix `model.layers.{i}.`), mapped from *unfused* HF names (vLLM fuses qkv_a + indexer wk/weights_proj and stacks experts; our loader maps the unfused form, reusing DeepSeek's `MLAEinsum` split of `kv_b_proj`→`k_up`/`v_up` and the expert-stacking transform):
- **Attention:** `self_attn.{q_a_proj, q_a_layernorm, q_b_proj, kv_a_proj_with_mqa, kv_a_layernorm, kv_b_proj, o_proj}.weight`
- **Indexer:** `self_attn.indexer.{wq_b, wk, weights_proj}.weight` + `self_attn.indexer.k_norm.{weight, bias}` (LayerNorm ⇒ has bias)
- **MoE:** `mlp.gate.weight` + `mlp.gate.e_score_correction_bias`; per-expert `mlp.experts.{j}.{gate_proj, up_proj, down_proj}.weight`; `mlp.shared_experts.{gate_proj, up_proj, down_proj}.weight`
- **Dense layers:** `mlp.{gate_proj, up_proj, down_proj}.weight`
- **Top-level:** `model.embed_tokens.weight`, `model.norm.weight`, `lm_head.weight` (untied)

## 6. Phase plan

### Phase 0 — CPU dev environment
**Deliverable:** a working JAX-CPU setup on the Mac plus an empty-but-runnable test harness.
- Create an isolated venv: `jax`, `jaxlib`, `flax`, `transformers`, `numpy`, `pytest`, `pytest-mock`. **Omit `libtpu`** (no macOS wheel; `requirements.txt:10` hard-pins `libtpu==0.0.41`). **Do NOT `pip install -e .`** (pulls the pin and AVX2 flag).
- On Apple Silicon, ensure `--xla_cpu_max_isa=AVX2` is **not** injected (`env_override.py:17`). Use `JAX_PLATFORMS=cpu` (sanctioned, `envs.py:208`).
- Stand up `tests/models/jax/test_glm_v52.py` (mirroring `test_deepseek_v3.py`): a **tiny GLM config** (≥1 dense + ≥1 MoE layer so both FFN paths are exercised, small hidden, few experts, small seq), an **HF-transformers-eager oracle** harness, and a **pure-`jnp` reference scaffold**.
- **Exit criterion:** `JAX_PLATFORMS=cpu python -m pytest tests/models/jax/test_glm_v52.py` collects and runs an import-clean smoke test.

### Phase 1 — Dense GLM (no DSA)
**Deliverable:** a complete, registered, config-driven `glm4_moe_lite.py` that is a correct dense-MLA GLM (full O(n²) attention, no indexer). De-risks ~80% of the model independent of DSA.
- Build the model reusing DeepSeek modules (§5), config-driven from `hf_config`, with the §4 router/activation deltas. No indexer wired.
- Loads via the registry with the §5 HF weight names (random tiny weights for CPU).
- **Validate per-submodule on CPU vs HF-eager** (`assert_allclose`, tolerances per §7):
  - Router: sigmoid + `e_score_correction_bias` + top-4 + `norm_topk_prob` + 1.8-on-output.
  - RMSNorms (q_a, kv_a, final), projections.
  - MoE with `USE_DENSE_MOE=1` (pure `jnp.einsum`, `dense_moe.py:51`).
  - MLA attention via a **pure-`jnp` MLA reference** computing the **non-absorbed** path (direct `kv_b_proj` expand of the latent into per-head K/V); the absorbed/weight-folded path is a TPU perf optimization deferred to Phase 3. (The production Pallas MLA kernel has no CPU/interpret mode.)
- **Exit criterion:** all CPU-runnable submodules match HF-eager within tolerance; full forward on the tiny config produces finite logits matching the pure-`jnp` reference.

### Phase 2 — DSA
**Deliverable:** the lightning-indexer module + top-k selection (V3.2 form), validated on CPU, plus the adapted Mosaic `sparse_attn` kernel ready for the TPU path.
- Add the `Indexer` (no ReLU/softmax, LayerNorm `k_norm`, `top-2048`, per-layer schedule with first ~2 layers dense) at the `compute_attention` hook.
- Pure-`jnp` reference mirroring the fork's `sparse_attn` CPU reference (`deepseek_v4_attention.py:179-216`), adapted V4→V3.2 (drop the ReLU/compressor).
- **Validation ladder:**
  1. **seq_len ≤ 2048:** assert sparse output == dense output (§2 equivalence) **and** the indexer's top-k contains all valid tokens.
  2. **seq_len > 2048:** assert the indexer selects the correct top-2048 tokens (vs the reference scorer) and sparse attention matches a masked-dense reference over the selected set.
- Port/adapt the fork's Mosaic `sparse_attn` kernel (`kernels/sparse_attn/kernel.py:117`) for the TPU path (wired in Phase 3).
- **Exit criterion:** both ladder checks pass on CPU against the pure-`jnp` reference.

### Phase 3 — TPU bring-up
**Deliverable:** single-sequence GLM 5.2 correctness on TPU.
- Wire the real Pallas/Mosaic kernels: extend `mla_ragged_paged_attention` (or add a sparse variant) to take gather indices; integrate the indexer-key KV cache; add `topk_indices` to `AttentionMetadata` (`attention_metadata.py:36`, built in `tpu_runner.py:2493`); target the `mla_attention` shard_map contract (`attention_interface.py:485`).
- Paged-KV integration and sharding (mesh axes per `sharding.py:32`).
- The pure-`jnp` references from P1/P2 become the **oracles** for kernel-vs-reference unit tests.
- **Exit criterion:** single-sequence generation on TPU matches the CPU reference within bf16 tolerance. (Perf + the fork's multi-batch MoE bug are addressed only after this, outside the v1 correctness gate.)

### Phase 4 — MTP (out of v1 scope)
Listed for completeness. `Glm4MoeLiteMTP` (`glm4_moe_lite_mtp.py`), gated by `num_nextn_predict_layers > 0`, weights filtered via `get_spec_layer_idx_from_weight_name`. Not built in v1.

## 7. Testing & validation strategy

**Reality being fixed:** today *no executing test compares JAX numerics vs an external reference* — existing model tests do shape checks, `assert is not None` smoke, and self-consistency; the one HF-eager comparison (`test_qwen3.py:351-356`) is dead code behind `if True: return`. The only running numeric parity is `test_weight_utils.py:112` (torch-MLP vs jax-MLP). This spec deliberately introduces real numeric parity for GLM.

- **Oracle:** HuggingFace-transformers-**eager** forward, with a **tiny random-weights** config (no checkpoint download). Same random weights loaded into both the HF reference and our JAX model.
- **CPU-runnable with numeric parity** (inline torch/HF ref + `assert_allclose`): router, RMSNorms, MoE (with `USE_DENSE_MOE=1`), projections, embedding/lm_head.
- **Guarded behind pure-`jnp` references** (no CPU kernel): MLA attention, DSA indexer + sparse attention. These references are the **reusable oracles** for TPU kernel validation in Phase 3.
- **Tolerances:** start at `rtol=1e-2, atol=1e-2` (matching the repo's existing HF-comparison intent) for full submodules; tighten to `rtol=1e-3, atol=1e-2` for isolated linear/MLP pieces (matching `test_weight_utils.py`). CPU runs fp32; expect to loosen for bf16 on TPU.
- **Test invocation:** `JAX_PLATFORMS=cpu USE_DENSE_MOE=1 python -m pytest -s -v tests/models/jax/test_glm_v52.py`.
- **Instantiation pattern:** `MockVllmConfig` + random weights + tiny hand dims (à la `test_deepseek_v3.py`), single-device mesh from `conftest.py` (`assert num_devices==1`). Note the existing DeepSeek attention test hard-requires a TPU (`TestDeepseekV3Attention.setUp:614`); our CPU tests must not inherit that requirement.

**What CPU correctness does and does not buy:** matching HF-eager on CPU kills the dominant new-model bug class (wrong weight mapping, router logic, MLA/MoE math, RoPE/head-split) — call it ~70–80% of "does it work" risk. It does **not** catch: Pallas/Mosaic kernel bugs, sharding/multi-device bugs, paged-KV/serving bugs, or bf16/fp8 precision drift. Those are chased on TPU in Phase 3, which is exactly why the pure-`jnp` references are written to be reused as kernel oracles.

## 9. Risks & mitigations

| Risk | Mitigation |
|---|---|
| Param-izing DeepSeek globals regresses the working DeepSeek model | Optional args defaulting to current behavior; keep existing DeepSeek tests green as a gate. |
| Indexer hyperparameters (`index_n_heads`, `index_head_dim`) differ between code defaults (64/128) and the real GLM-5.2 checkpoint (possibly 32) | Read all indexer dims from `hf_config`; never hardcode. Validate against the real checkpoint config when available. |
| `first_k_dense_replace` absent from the HF Lite dataclass | Read `config.first_k_dense_replace` from the checkpoint (as vLLM does); fail loudly if missing. |
| Accidentally copying the fork's V4 indexer (ReLU + compressor) | §4 calls this out explicitly; the Phase 2 reference is V3.2 (no ReLU, LayerNorm k_norm). |
| CPU parity passes but TPU fails | Expected and bounded — §7 enumerates the TPU-only risk classes; pure-`jnp` references become kernel oracles in Phase 3. |
| bf16 tie-flips in router top-k / indexer top-k on TPU | Phase 3 tolerance loosening + tie-aware comparison; CPU fp32 establishes the ground truth first. |
